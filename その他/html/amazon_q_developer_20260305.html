01 OVERVIEW

Amazon Q Developerとは

CodeWhispererから進化したAWS公式のAIコーディングアシスタント。 ソフトウェア開発ライフサイクル全体をカバーする統合AI基盤です。

🎯 概要

Amazon Q Developerは、AWSが提供する生成AI搭載の開発アシスタントです。コードの記述、デバッグ、テスト、セキュリティスキャン、レガシーアプリケーションのアップグレードまで、ソフトウェア開発ライフサイクル（SDLC）全体をAIで支援します。IDE内でのコード補完やチャットはもちろん、AWSコンソール上でのトラブルシューティング、コマンドラインでのエージェント操作まで、あらゆるタッチポイントで開発者を助けるプラットフォームです。

2024年4月30日、従来のAmazon CodeWhispererがAmazon Q Developerへとリブランドされ、コード補完機能に加えて、チャット、エージェント、コード変換、セキュリティスキャンなど多数の新機能が統合されました。これはAWSが「AIファースト」の開発者体験を実現するための戦略的な大型刷新であり、単なる名称変更ではなく、製品の能力が大幅に拡張されたことを意味します。

Amazon Q DeveloperはAmazon Bedrock上に構築されており、高い安全性と責任あるAI利用の管理機能を継承しています。25以上のプログラミング言語に対応し、Python、Java、JavaScript、TypeScript、C#、Go、Rust、PHP、Ruby、Kotlin、C、C++、Shell、SQL、Scala、JSON、YAML、HCLなどをサポートします。

CodeWhispererからの進化の歴史
2022年 — 誕生

Amazon CodeWhisperer プレビュー開始

AWSがAIコード生成サービスとして「Amazon CodeWhisperer」をプレビューリリース。リアルタイムのコード補完を中核機能として、GitHub Copilotに対抗するAWS公式ツールとして登場しました。

2023年4月 — 一般提供

CodeWhisperer GA + 個人向け無料化

一般提供（GA）が開始され、個人開発者向けに無料で利用可能になりました。セキュリティスキャン機能やリファレンストラッキング機能も追加。VS CodeとJetBrains IDEへの対応が強化されました。

2024年4月 — 大型進化

Amazon Q Developerへリブランド

CodeWhispererの全機能がAmazon Q Developerへ統合されました。チャット機能、エージェント機能（/dev）、コード変換機能（/transform）、ドキュメント生成、コードレビューなど大幅な機能拡張が実施されました。同時に、Amazon Qファミリー（Q Business、Q Developer等）の一部として位置づけられました。

2024年12月 — re:Invent

エージェント機能の大幅強化

re:Invent 2024にて、ドキュメント生成、コードレビュー、ユニットテスト自動生成などの新しいエージェント機能が発表。Amazon Q Developerの「エージェント指向」開発が加速しました。

2025年3月〜 — CLI強化

CLIエージェント・MCP対応

コマンドラインでのエージェント体験が刷新され、自然言語での指示からファイルの読み書き、AWSリソースのクエリ、コード生成が可能になりました。4月にはModel Context Protocol（MCP）対応も発表。

2025年5月〜 — アジェンティック

IDE内アジェンティック体験の全面展開

VS Code、JetBrains、Visual StudioでのフルAgenticコーディング体験が提供開始。自然言語からの機能実装、マルチファイル変更、テスト自動実行など、開発者が「指示するだけ」で成果物が生成される新しい開発パラダイムが実現しました。

2025年12月 — re:Invent 2025

クラウド開発の全面AI化

re:Invent 2025にて、コスト管理機能の強化、アーキテクチャ設計支援、CloudWatchログ・メトリクスの自動分析、Kiro IDEとの連携などが発表されました。Amazon Q Developerは開発だけでなく、運用・監視・最適化までカバーする総合AIプラットフォームへと進化しています。

02 FEATURES

主な機能

コード補完からエージェント開発まで、SDLCのあらゆるフェーズをカバーする5大機能群を解説します。

✍️

インラインコード補完

リアルタイムでコードスニペットから関数全体まで生成。25以上の言語に対応し、既存コードとコメントの文脈を理解して最適な提案を行います。BT Groupは37%、NABは50%のコード受入率を報告しています。

💬

AIチャット

IDE内やCLI、AWSコンソールで自然言語の質問に回答。プログラムロジックの解説、バグの特定、AWSサービスのベストプラクティス相談など、コンテキストを理解した対話が可能です。多言語（日本語含む）対応。

🤖

エージェント（Agentic Coding）

自然言語のプロンプトから、ステップバイステップの実装計画を自動生成。コードの作成・テスト・APIインテグレーションを一括実行し、マルチファイルにまたがる機能実装を数分で完了します。

🛡️

セキュリティスキャン

SAST（静的アプリケーションセキュリティテスト）によりコード内の脆弱性を検出。SQLインジェクション、リソースリーク、シークレット漏洩、暗号化の不備などを自動的にスキャンし、修正提案まで行います。

🔄

コード変換（Transform）

JavaアプリケーションのJDKバージョンアップグレード（Java 8/11 → 17/21）や.NETアプリのWindows→Linux移植をAIエージェントが自動実行。依存関係の更新、非推奨API対応、ビルド確認まで自律的に行います。

📝

コードレビュー & ドキュメント生成

コードの品質問題（論理エラー、アンチパターン、コード重複）を自動レビュー。データフロー図を含むドキュメントを自動生成し、READMEの更新もQ Developerに依頼できます。

🧪 ユニットテスト自動生成

Amazon Q Developerはコードの選択範囲やプロジェクト全体に対してユニットテストを自動生成します。テストフレームワーク（JUnit、pytest、Jest等）に準拠したテストコードを作成し、エッジケースやモック設定も含めた実用的なテストを提供します。「このメソッドのテストを書いて」と依頼するだけで、IDE内にテストコードが即座に生成されます。

🖥️ Console-to-Code

AWSコンソール上で行った操作をキャプチャし、それをCloudFormation、CDK、AWS CLIなどの再利用可能なコードに変換する機能です。プロトタイピングから本番デプロイメントへの移行を高速化します。コンソールでの試行錯誤をそのまま本番用のInfrastructure as Code（IaC）に転換できるため、手動設定からコードベースの運用への移行が容易になります。

🔗 MCP（Model Context Protocol）対応

2025年4月よりCLIで、6月よりIDE プラグインでMCPサポートが追加されました。MCPはAIモデルが外部ツール、データソース、APIにセキュアにアクセスするためのオープンプロトコルです。これにより、Amazon Q Developerはデータベーススキーマの理解、外部サービスとの連携、プロジェクト固有のコンテキスト取得が可能になり、より精度の高いコード生成やアドバイスが実現します。

⚙️ カスタマイズ機能

Amazon Q Developerは、組織の内部ライブラリ、API、ベストプラクティス、アーキテクチャパターンを学習させることで、より関連性の高いコード推奨とチャット応答を生成できます。Pro Tierではこのカスタマイズ機能を活用でき、社内のコードベースに合わせた提案が可能です。対応言語はPython、Java、JavaScript、TypeScript、C#、C++で、チームの生産性向上に直結します。さらに「Rules」機能により、コーディング標準やベストプラクティスを定義し、Q Developerのコンテキストとして自動適用させることもできます。

🧠 Amazon Bedrock 基盤

Amazon Q DeveloperはAmazon Bedrock上に構築されています。Bedrockは、AWSが提供するフルマネージドの基盤モデル（FM）プラットフォームであり、Anthropic、Meta、Amazon自社製モデルなど多数のFMをAPI経由で利用可能にするサービスです。Q Developerはこの基盤を活用することで、エンタープライズ級のセキュリティ、データプライバシー、スケーラビリティを実現しています。

Bedrock上に構築されているため、自動的に悪用検出メカニズムが組み込まれており、安全性、セキュリティ、責任あるAI利用が確保されています。ユーザーのコードがモデルのトレーニングに利用されることはなく（Pro Tierでは自動的にオプトアウト）、データの機密性が保護されます。

🔬 使用モデルについて

Amazon Q Developerの料金ページには「Access latest Claude models（最新のClaudeモデルにアクセス）」と明記されており、Anthropic社のClaudeモデルが中核的な推論エンジンとして活用されていることがわかります。ただしAWSは、Q Developer内部で使用される具体的なモデル構成の詳細を完全には公開していません。コード補完、セキュリティスキャン、コード変換などの各機能では、それぞれに最適化されたモデルやパイプラインが使い分けられていると考えられます。

また、AWSはCodeGuruの技術を継承したセキュリティ検出器ライブラリを保有しており、セキュリティスキャン機能ではこれらの特化型検出エンジンと生成AIの組み合わせによる高精度な脆弱性検出が実現されています。

⌨️ CLIエージェントの特徴

Amazon Q Developer CLIは、ターミナルに直接AIアシスタントを統合するツールです。主な機能として、コマンド自動補完（コマンドやオプションのインテリジェントな提案）、自然言語→bashコマンド変換（「S3の全バケットを一覧して」などの日本語や英語の指示をコマンドに変換）、エージェント機能（ファイルの読み書き、AWSリソースへのクエリ、コードの生成を自然な対話で実行）があります。

💰 料金の詳細・注意点

アクティベーション方式: Pro Tierは「サインインしただけ」では課金されません。エージェントコーディングのプラン生成依頼、コード変換の実行、コード補完の利用など、実際にQ Developerの機能を使用した時点でサブスクリプションが有効化されます。初月は日割り計算され、以降は月額$19が自動更新されます。

Java変換の課金体系: Pro Tierでは月4,000行/ユーザー分が含まれ、これはAWSペイヤーアカウントレベルでプール集計されます。例えば10ユーザーのPro契約なら月40,000行までが含まれ、超過分は$0.003/行の従量課金となります。途中停止やエラー終了した変換、コメントや空行は課金対象外です。

月途中のキャンセル: 月途中でサブスクリプションをキャンセルした場合、その月分は全額課金されます。キャンセル即座にアクセス権が失われるため、月末でのキャンセルが推奨されます。

Builder IDからのアップグレード: 2025年6月より、Builder IDユーザーもPro Tierへアップグレード可能になりました。ただし、Builder IDでPro Tierにアップグレードした場合は、管理ダッシュボード機能は利用できません。管理機能が必要な場合はIAM Identity Centerでのサインアップが必要です。

🎯
個人開発者へのおすすめ: まずはFree Tierで月50回のエージェントリクエストを試し、価値を感じたらPro Tierへのアップグレードを検討しましょう。AWS利用がメインの開発者にとっては、$19/月は十分にペイする投資です。
06 AWS INTEGRATION

AWS特化機能

Lambda、CloudFormation、CDKなど、AWSサービスとの深い統合がQ Developerの最大の差別化ポイントです。

⚡

Lambda関数生成

自然言語の説明からLambda関数のコードを自動生成。イベントソース設定、権限設定、エラーハンドリングまで含めた実用的なコードをIDE内で作成します。Python、Node.js、Java、Go等に対応。

📋

CloudFormation支援

CloudFormationテンプレート（YAML/JSON）の生成・修正をAIで支援。リソース定義、パラメータ設計、出力設定を自然言語で指示してテンプレートを作成できます。カスタムリソースの作成支援も可能です。

🏗️

CDK（Cloud Development Kit）支援

TypeScript、Python等でのCDKコンストラクト生成を支援。CDKテストの自動生成、CDK v1→v2マイグレーション、3階層Webアプリケーションのインフラ構築までAIでサポートします。

🔍

AWSコンソールエラー診断

AWSコンソール上でエラーが発生した際、Q Developerが自動的にエラーを分析し、権限不足、設定ミス、サービス制限などの根本原因と解決策を提示します。全AWSコマーシャルリージョンで利用可能。

💸

コスト管理・最適化

AWSのコスト分析をQ Developerが支援。自然言語でサービス料金の詳細やワークロードコストの見積もりを取得でき、コスト最適化の推奨事項も提供されます（2025年10-11月に大幅強化）。

📊

CloudWatch / ログ分析

CloudWatchのログ、メトリクス、ダッシュボードを自動分析し、パフォーマンスの問題やエラーの原因を特定します。ECSタスクやEC2インスタンスのトラブルシューティングにも対応。

🌐

ネットワークセキュリティ分析

WAFやネットワーク設定のセキュリティ分析をAIで実行。セキュリティインサイトの提供と具体的な修正オプションを提示し、クラウドインフラの防御強化を支援します。

🎨

ホワイトボード→クラウド変換

MCP＋Bedrock Data Automationを活用して、ホワイトボードに描いたアーキテクチャ図を実際のクラウドインフラコードに変換。構想から実装までの時間を劇的に短縮します。

07 SECURITY

セキュリティスキャン・脆弱性検出

SAST、シークレット検出、IaCスキャンを統合したコードセキュリティプラットフォーム。

🔐 セキュリティスキャンの全体像

Amazon Q Developerのセキュリティスキャン機能は、開発ライフサイクルの複数段階でコードのセキュリティを検証する包括的なソリューションです。SAST（静的アプリケーションセキュリティテスト）によるソースコード内の脆弱性検出、シークレット検出によるハードコードされた認証情報やAPIキーの発見、IaC（Infrastructure as Code）スキャンによるCloudFormation・Terraform等のセキュリティ設定検証の3つの柱で構成されています。

これらの検出機能の背後には、Amazon CodeGuru Reviewerから継承された検出器ライブラリ（Detector Library）があり、数百種類のセキュリティパターンとコード品質パターンが登録されています。生成AIの推論能力とルールベースの検出器を組み合わせることで、高い精度と網羅性を実現しています。

セキュリティスキャンは、IDE内でオンデマンドまたはコードレビュー時に実行できます。検出された脆弱性には深刻度レベルが付与され、多くの場合は具体的な修正コードが提案されます。GitHub連携時にはPull Request作成時にも自動スキャンが実行されるため、セキュリティ問題をコードレビュー段階で発見・修正できます。Amazon Inspectorとの連携により、デプロイ後の実行時脆弱性スキャンも統合的に管理可能です。

Java / .NET アップグレード変換機能

☕ Java アップグレード

Amazon Q Developerのコード変換機能は、Javaアプリケーションのバージョンアップグレードを自動化します。Java 8 → 17/21、Java 11 → 17/21 のアップグレードに対応し、非推奨API の置き換え、ライブラリ・フレームワーク・依存関係の更新、ビルド確認、テスト実行まで自律的に行います。

2025年6月にはSelective Transformation機能がCLI（Linux/macOS）でプレビュー開始。特定のサードパーティ依存関係のみを選択的にアップグレードするきめ細かい制御が可能になりました。また、組み込みSQL変換にも対応し、データベースアクセス層の近代化も支援します。

🔷 .NET ポーティング

.NETアプリケーションのWindows→Linux移植を最大4倍高速化する変換機能です。.NET Framework から .NET（クロスプラットフォーム対応版）への移行を支援し、Windows固有のAPIの置換、ファイルパスの正規化、プラットフォーム依存コードの変換を自動実行します。

変換完了後には、変更ファイル数、更新パッケージ数、変更API数などの変換サマリーが提供され、Linux互換性の検証に役立つ情報が得られます。

🔍 比較のポイント

AWSエコシステムの開発者には、Amazon Q Developerが最適です。AWS サービスとのネイティブ統合（コンソールエラー診断、CloudWatch分析、コスト管理、Console-to-Code）は他のツールでは実現できない独自の価値であり、AWS上でアプリケーションを構築・運用するチームにとって生産性を大きく向上させます。

GitHub Copilotは最も広く普及しているAIコーディングアシスタントで、インライン補完の速度と精度に定評があります。GitHubとの密接な連携がメリットですが、クラウド特化の機能はQ Developerに比べて限定的です。

CursorはVS Codeをフォークした独自IDEであり、マルチファイル編集やComposer機能でのUI/UX体験に優れています。フロントエンド開発やラピッドプロトタイピングにおいて高い評価を得ていますが、クラウドインフラ管理やセキュリティスキャンの機能は持ちません。

Claude CodeはAnthropicのCLIベースのコーディングエージェントで、複雑なコードベースの理解と大規模なリファクタリングに強みがあります。IDE非依存でターミナルから直接操作でき、深い推論能力でコード品質の高い成果物を生成します。

OpenAI Codexはクラウドベースのソフトウェアエンジニアリングエージェントで、複数タスクの並列実行が可能です。2026年2月にはmacOS用のCodex Appもリリースされ、エコシステムが拡大しています。

各ツールは異なる強みを持つため、プロジェクトの技術スタック、チームの開発ワークフロー、クラウドプラットフォームの選択に応じて最適なツールが変わります。複数のツールを併用するチームも増えています。

12 GETTING STARTED

始め方ガイド・参考リンク

今日からAmazon Q Developerを使い始めるためのステップバイステップガイド。

🚀 VS Codeでの始め方（5ステップ）

ステップ1: VS Codeの拡張機能マーケットプレイスで「Amazon Q」を検索し、AWS公式の「Amazon Q」拡張機能をインストールします。これはAWS Toolkitの一部として提供されています。

ステップ2: インストール後、サイドバーにAmazon Qのアイコンが表示されます。クリックして「Use for Free」または「Sign in」を選択します。無料で始める場合は、AWS Builder IDを作成します（メールアドレスだけで登録可能、AWSアカウント不要）。

ステップ3: 認証が完了すると、即座にインラインコード補完が有効になります。コードを書き始めると、グレーのゴースト表示で補完候補が表示されます。Tabキーで受け入れ、Escキーで拒否します。

ステップ4: サイドバーのチャットパネルからQ Developerに質問できます。「この関数を最適化して」「S3にファイルをアップロードするLambda関数を書いて」など、自然言語で指示を出しましょう。日本語での質問にも対応しています。

ステップ5: エージェント機能を試すには、チャットで「/dev」と入力し、実装したい機能を自然言語で説明します。Q Developerが実装計画を提案し、承認するとコードの生成・変更を自動実行します。

⌨️ CLIでの始め方

macOS: brew install amazon-q でインストールできます。インストール後、q login でBuilder IDまたはIAM Identity Centerで認証を行います。

Linux: 公式サイトからインストーラをダウンロードするか、パッケージマネージャ経由でインストールします。

認証後は、ターミナルで自動補完が有効になります。q chat でチャットモード、q でインタラクティブなエージェントセッションを開始できます。「AWSのS3バケット一覧を表示して」のような自然言語での指示が可能です。
