Section 01 — Overview

Codexとは何か

OpenAI Codex は、OpenAIが開発したクラウドベースのAIソフトウェアエンジニアリングエージェントです。2025年5月に初めてリサーチプレビューとして公開され、その後急速に進化を遂げました。従来の「コード補完ツール」とは根本的に異なり、Codexは自律的にコードを読み、書き、実行し、テストまで行える完全なエージェントシステムです。

人間の開発者のように、リポジトリ全体を理解し、バグ修正・新機能実装・コードレビュー・テスト作成・プルリクエスト提出までを一貫して処理できます。しかも、複数のタスクを並行して処理できるため、開発者は朝にタスクを割り当て、完了を待つ間に別の作業に集中できます。

💻
Codex Web（クラウド）

ChatGPTのサイドバーからアクセスできるクラウド版。各タスクが独立したサンドボックス環境で実行され、リポジトリを自動的に読み込みます。タスク完了後にGitHub PRを作成したり、ローカルに変更を取り込むことができます。

⌨
Codex CLI（ターミナル）

ターミナルで動作するオープンソース（Apache-2.0）のコーディングエージェント。Rustで構築されており高速かつ軽量。ローカルのリポジトリを直接読み書きし、コマンドを実行できます。

🖥
Codex App（デスクトップ）

2026年2月にリリースされたmacOS/Windows対応のデスクトップアプリ。複数のエージェントをプロジェクト単位で管理し、マルチエージェントワークフロー、オートメーション、スキル管理などの高度な機能を提供します。

🔧
IDE拡張機能

VS Code、Cursor、Windsurfなどの主要IDEで利用可能な拡張機能。エディタ内でCodexとペアプログラミングでき、インラインのdiff表示やプロンプト入力が可能です。JetBrains、Xcodeにも対応。

💡 重要な位置づけ

Codexは「GitHub Copilotの上位互換」ではなく、異なるカテゴリのツールです。Copilotが「スペルチェッカー」だとすれば、Codexは「自律的に論文を書ける研究アシスタント」のような存在です。Copilotはリアルタイムのコード補完に特化していますが、Codexは数分〜数時間かかる複雑なタスクを独立して遂行できます。

Section 02 — CLI Basics

Codex CLIの基本的な使い方

Codex CLIは、ターミナルから直接AIコーディングエージェントを操作できるオープンソースツールです。macOSとLinuxで正式サポートされており、Windowsは WSL（Windows Subsystem for Linux）経由で利用できます。

インストール方法

3つのインストール方法から選べます。

npm 推奨 
# npmでグローバルインストール
$ npm install -g @openai/codex
Homebrew macOS / Linux 
# Homebrewでインストール
$ brew install --cask codex
Binary 手動ダウンロード 
# GitHubリリースページからバイナリをダウンロード
# macOS (Apple Silicon): codex-aarch64-apple-darwin.tar.gz
# macOS (Intel):         codex-x86_64-apple-darwin.tar.gz
# Linux (x86_64):        codex-x86_64-unknown-linux-musl.tar.gz
# Linux (arm64):         codex-aarch64-unknown-linux-musl.tar.gz
$ tar xzf codex-*.tar.gz
$ mv codex-* /usr/local/bin/codex

認証とログイン

インストール後、codex コマンドを実行すると、初回起動時に認証を求められます。ChatGPTアカウントでサインインする方法が推奨されています。

Shell 初回起動 
# Codexを起動（初回は認証プロンプトが表示される）
$ codex

# 「Sign in with ChatGPT」を選択
# → ブラウザが開き、ChatGPTアカウントでログイン
# → APIキーが自動設定される
Shell APIキーで認証する場合 
# 環境変数でAPIキーを設定
$ export OPENAI_API_KEY="sk-your-api-key-here"

# または config.toml に設定
# ~/.codex/config.toml

承認モード（Approval Modes）

Codex CLIには3つの承認モードがあり、セキュリティと利便性のバランスを調整できます。

モード ファイル編集 コマンド実行 推奨シーン
Suggest（提案） ✅ 承認が必要 ✅ 承認が必要 初めて使う場合、セキュリティ重視のプロジェクト
Auto-edit（自動編集） ✔ 自動承認 ✅ 承認が必要 日常的な開発作業、バランス重視
Full-auto（完全自動） ✔ 自動承認 ✔ 自動承認 信頼できるプロジェクト、スクリプト自動化
Shell 承認モードの指定 
# 完全自動モードで起動
$ codex --ask-for-approval never "テストを全て実行して結果を報告して"

# YOLOモード（サンドボックス内で完全自動）
$ codex --yolo

アップグレード

Shell 最新版にアップグレード 
# npmの場合
$ npm i -g @openai/codex@latest

# Homebrewの場合
$ brew upgrade --cask codex
Section 03 — Features

主な機能と特徴

🔗
GitHub連携

リポジトリを自動でクローン・解析し、PRの作成・コードレビューの実行まで対応。@Codex でPR上のレビューを直接依頼できます。

🔮
マルチエージェント

複数の専門エージェントを並列に生成し、探索・レビュー・実装などを分担。CSV一括処理（spawn_agents_on_csv）にも対応しています。

📜
AGENTS.md

リポジトリにAGENTS.mdファイルを配置することで、Codexにプロジェクト固有の指示（テストコマンド、コーディング規約等）を伝達できます。

🔒
セキュアサンドボックス

各タスクは隔離されたコンテナ環境で実行。クラウド版ではデフォルトでネットワークアクセスが無効化され、安全にコードを実行できます。

🎨
Skills（スキル）

Figmaデザイン実装、Linear管理、Cloudflare/Vercelデプロイ、画像生成、ドキュメント作成など、拡張可能なスキルシステムでCodexの能力を拡張。

🔄
Automations

スケジュールベースで自動的にタスクを実行。日次のイシュートリアージ、CI失敗の要約、リリースブリーフ生成などを自動化できます。

💬
Slack連携

Slackチャンネルから @Codex でタスクを投げると、クラウドタスクを作成してスレッドに結果を返します。チームのワークフローにシームレスに統合可能。

🌐
Web検索

Codex CLIからWebを検索して最新の情報を取得可能。ライブラリのドキュメントやAPIリファレンスを参照しながらコーディングできます。

🖼
画像入力

スクリーンショットやデザインスペックを添付して、UIの実装を指示できます。Figmaスキルと組み合わせることで高精度なUI再現が可能。

🔌
MCP対応

Model Context Protocol (MCP) によりサードパーティツールと連携。Chrome DevTools、ドキュメントサーバー、カスタムツールを統合できます。

👥
エージェントロール

explorer（探索）、worker（実装）、reviewer（レビュー）、monitor（監視）など、用途別のエージェントロールを定義・カスタマイズ可能。

👥
インタラクティブ操作

GPT-5.3-Codexでは、作業中のエージェントにリアルタイムでフィードバックや方向修正が可能。進捗の定期報告も受けられます。

AGENTS.md の仕組み

AGENTS.md は、Codexにプロジェクト固有のルールを伝えるためのMarkdownファイルです。Codexは起動時（セッション開始時）にこのファイルを読み込み、指示に従って動作します。グローバル設定（~/.codex/AGENTS.md）からプロジェクトルート、さらにサブディレクトリへと階層的に読み込まれ、より近い位置のファイルが優先されます。

Markdown AGENTS.md の例 
# AGENTS.md

## プロジェクトルール

- テスト実行コマンド: `npm test`
- Linter: `npm run lint` をPR前に必ず実行
- TypeScriptを使用、any型は禁止
- コミットメッセージは Conventional Commits 形式で
- 新しい依存関係の追加前に確認を求めること

## ディレクトリ構成

- `src/` - メインソースコード
- `tests/` - テストファイル
- `docs/` - ドキュメント

## コーディング規約

- 関数には JSDoc コメントを付ける
- エラーハンドリングは必ず行う
- マジックナンバーは定数に切り出す
TOML ~/.codex/config.toml — マルチエージェント設定例 
# マルチエージェントを有効化
[features]
multi_agent = true

# エージェント設定
[agents]
max_threads = 6
max_depth = 1

[agents.explorer]
description = "コードベース探索用の読み取り専用エージェント"
config_file = "agents/explorer.toml"

[agents.reviewer]
description = "正確性・セキュリティ・テストに焦点を当てたPRレビュアー"
config_file = "agents/reviewer.toml"
Section 04 — Models

GPT-4o / GPT-5.3との関係

OpenAI Codexの内部で使用されるモデルは、リリース以来急速に進化しています。Codex専用にファインチューニングされた「Codex系モデル」が存在し、汎用のGPTモデルとは区別されます。以下にその進化の歴史を示します。

モデル進化タイムライン

2021年8月
初代 Codex（GPT-3ベース）
GitHub Copilotのバックエンドとして使われた、最初の「Codex」ブランド。自然言語からコードを生成する能力でAIコーディング革命の先駆けとなりました。現在は非推奨。
2025年4月
Codex CLI リリース（オープンソース）
ターミナルで動作する軽量コーディングエージェント。o3やo4-miniモデルを使用して、ローカル環境でのペアプログラミングを実現。

codex-1 + codex-mini（o3ベース）
Codex Web（クラウド版）の初回リリース。codex-1はo3をソフトウェアエンジニアリング向けに最適化したモデル。codex-miniはo4-miniベースの高速版。強化学習により人間のコーディングスタイルに近い出力を実現。

GPT-5をコーディングタスク向けに最適化。SWE-Bench Verifiedで51.3%を達成し、ベースモデル（33.9%）を大幅に上回る。Responses APIで利用可能に。

長時間エージェントタスクに最適化。Maxバージョンは特に大規模プロジェクト向け。Miniは4倍の利用効率を提供。

現時点で最も強力なCodexモデル。SWE-Bench Proで56.8%、Terminal-Bench 2.0で77.3%を達成。自身のトレーニングとデプロイに使用された初のモデル。25%高速化。OSWorldで64.7%（人間は約72%）。

Section 05 — Pricing

料金体系・利用条件

Codexは ChatGPT のサブスクリプションに含まれており、追加料金なしで利用を開始できます。2026年3月現在、期間限定で Free および Go プランにも Codex が開放されており、有料プランでは2倍のレート制限が適用されています。

メッセージ数の幅は、タスクの規模・複雑さによって変動します。小さなスクリプトは少量で済みますが、大規模なコードベースの長時間タスクはより多くの制限を消費します。

クレジットシステム

利用制限に達した後も、追加クレジットを購入して作業を継続できます。クレジット消費量はモデルと機能によって異なります。

💰 Pro限定特典

Proプラン（$200/月）では、GPT-5.3-Codex-Spark（超高速モデル）へのアクセス、6倍のローカル/クラウド制限、10倍のコードレビュー数、優先リクエスト処理が含まれます。本格的な日常開発に Codex を使う場合に最適です。

⚠ 利用制限を長持ちさせるコツ

プロンプトを簡潔にする、AGENTS.md のサイズを最適化する、不要な MCP サーバーを無効化する、日常的なタスクには GPT-5.1-Codex-Mini を使用する（約4倍の効率）— これらの工夫で同じ予算でより多くの作業が可能になります。

Section 06 — Use Cases

ユースケース（何に使えるか）

Codexは単なるコード生成ツールではなく、ソフトウェア開発ライフサイクル全体をカバーするエージェントです。GPT-5.3-Codexにより、コーディング以外のナレッジワークにもその能力が拡張されています。

🛠
新機能の実装

「ユーザー認証機能をOAuth2で実装して」のようなプロンプトで、ファイル作成からテスト記述、PRの作成まで一貫して実行。複数のエージェントを並列に走らせて異なる機能を同時に開発することも可能です。

🐛
バグの修正

エラーログやスタックトレースを渡すと、原因を特定し修正パッチを生成。テストを実行して修正が正しいことを検証してからコミットします。オンコール対応の効率化にも活用されています。

🔍
コードレビュー

GitHub PRで @Codex をメンションすると、セキュリティ問題、バグ、コード品質、テスト不足などを自動でレビュー。手動レビューの前のファーストパスとして非常に有効です。

🔁
リファクタリング

大規模なコードベースのリネーム、構造変更、デザインパターンの適用を安全に実行。テストを走らせながら破壊的変更がないことを確認します。

📝
テスト作成

既存コードのテストカバレッジを向上。ユニットテスト、統合テスト、E2Eテストの作成と実行を自動化し、カバレッジ率を報告します。

📄
ドキュメント生成

コードからAPIドキュメント、README、技術仕様書を自動生成。コードの変更に合わせてドキュメントを更新する自動化ワークフローも構築可能です。

🎨
UIデザインの実装

Figmaデザインのスクリーンショットを渡して「これを実装して」と指示するだけでプロダクションレディなUIコードを生成。画像生成スキルと組み合わせてアセットも自動作成。

🚀
デプロイ・CI/CD

Cloudflare、Vercel、Netlify、Renderへのデプロイをスキルで自動化。CI障害の診断・修正も実行可能。GPUクラスターの動的スケーリングにも使用されています。

📊
データ分析・レポート

CSV/スプレッドシートの分析、可視化、レポート生成。PDF・PowerPoint・Wordドキュメントの作成。GDPvalベンチマークで70.9%を達成し、知識労働タスク全般に対応。

🎮
ゲーム・アプリ開発

OpenAIのデモでは、1つのプロンプトからレーシングゲームやダイビングゲームを自律的に開発。7百万トークン以上を使って、デザイン・開発・QAテストまで一人で遂行。

🎯 実績のある企業での活用

Cisco（プロダクトポートフォリオ全体での評価）、Temporal（機能開発・デバッグ・リファクタリング）、Superhuman（テストカバレッジ・統合テスト・PMによるライトなコード変更）、Kodiak（自動運転技術のデバッグツール・テストカバレッジ向上）など、多くの企業で実運用されています。

Section 09 — Tips & Caveats

Tips・注意点

効果的に使うためのTips

🎯 プロンプトの書き方

Codexは明確で具体的な指示に対して最高のパフォーマンスを発揮します。「良いコードを書いて」ではなく「src/auth/login.tsにJWTベースの認証ミドルウェアを追加して。エラーハンドリングを含め、テストも書いて」のように、対象ファイル・技術スタック・期待する結果を明記してください。

複数の独立したタスクは別々のプロンプトとして送信し、並列に処理させましょう。1つの巨大なプロンプトより、スコープが明確な小さなタスクの方が成功率が高くなります。

📈 モデルの使い分け

日常的な小さなタスク（リネーム、フォーマット、簡単な修正）には GPT-5.1-Codex-Mini を使い、利用制限を節約しましょう。複雑なアーキテクチャ変更や長時間タスクには GPT-5.3-Codex を使用し、Proプランなら GPT-5.3-Codex-Spark でリアルタイムの高速イテレーションが可能です。/model コマンドでセッション中にモデルを切り替えられます。

🛠 AGENTS.md を活用する

AGENTS.md が適切に設定されているプロジェクトでは、Codexの出力品質が格段に向上します。テストコマンド、Linterの使い方、コーディング規約、ディレクトリ構成を記載しましょう。グローバル設定（~/.codex/AGENTS.md）に個人的な好みを書き、プロジェクトごとの設定で上書きする構成が効果的です。ファイルサイズはデフォルト32KBまで（config.tomlで変更可能）。

👥 マルチエージェントの活用

PRレビューでは、explorerでコードパスをマッピング → reviewerでバグ・セキュリティチェック → docs_researcherでAPI確認、というように役割を分けると効果的です。CSV一括処理（spawn_agents_on_csv）で大量の繰り返しタスクを並列化することも可能です。各エージェントのモデルやサンドボックス設定をロールごとにカスタマイズできます。

🔄 自動化ワークフロー

codex exec コマンドを CI/CD パイプラインやシェルスクリプトに組み込むことで、反復的なタスクを自動化できます。Codexアプリの Automations 機能では、スケジュールベースで定期的にタスクを実行し、結果をレビューキューに送れます。

注意点・よくあるトラブル

⚠ 必ずコードレビューを行う

Codexが生成したコードは必ず人間がレビューしてから本番環境にデプロイしてください。ハルシネーション（存在しないAPIの使用、誤ったロジック）が発生する可能性があります。特にセキュリティに関わる部分は慎重に確認してください。

⚠ サンドボックスの信頼性

Codex CLIのサンドボックスは基本的に安全ですが、過去にzshバイパスの脆弱性が報告されたことがあります（v0.106.0で修正済み）。--dangerously-bypass-approvals-and-sandbox フラグは信頼できる環境でのみ使用してください。本番サーバーでの使用は避けましょう。

⚠ 出力のばらつき

Codexは同一プロンプトでも異なる結果を返すことがあります。決定的な出力が必要な場合は、具体的なファイル名、関数名、期待する振る舞いを明記してください。対照的に、Claude Codeは同じプロンプトに対してより一貫した結果を返す傾向があります。

⚠ Windows環境での注意

Codex CLIのWindows対応は実験的です。最も安定した体験を得るには、WSL（Windows Subsystem for Linux）環境で使用してください。デスクトップアプリは2026年3月にWindows版がリリースされました。

💡 トークンと制限の管理

/status コマンドで残りの利用制限をリアルタイムに確認できます。利用状況ダッシュボード（chatgpt.com/codex/settings/usage）でも詳細を確認可能。MCP サーバーの数を減らすとコンテキストサイズが小さくなり、利用制限が長持ちします。
