Section 01

Cursorとは — 概要・Anysphere社・VS Code forkの経緯

AIをコードエディタの中核に据え、ソフトウェア開発の在り方を根本から変えようとするプロダクト

Cursorは、Anysphere, Inc.が開発・提供するAI搭載のコードエディタです。MicrosoftのオープンソースエディタVisual Studio Code（VS Code）をフォークし、AIによるコード補完、チャットベースの質問応答、マルチファイル編集、エージェントモードなど、AI機能をエディタの根幹に組み込んだ"AIネイティブIDE"として設計されています。単なる拡張機能の追加ではなく、エディタそのものの動作にAIを統合しているため、VS Codeの拡張機能では実現困難なレベルの深い統合を実現している点が最大の特徴です。

Anysphere社について

Anysphereは、2022年にMIT（マサチューセッツ工科大学）の卒業生4名—Michael Truell、Sualeh Asif、Arvid Lunnemark、Aman Sanger—によって設立されたサンフランシスコ拠点のAIスタートアップです。当初は「応用AIリサーチラボ」として始動し、その後CursorをメインプロダクトとしてAIコーディングツール市場を牽引する存在に成長しました。

驚異的な成長スピードもAnysphereの特筆すべき点です。2024年12月にThrive Capital主導で1億500万ドルのシリーズBを調達し評価額は26億ドル。2025年5月にはさらに9億ドルを調達して評価額99億ドルに。そして2025年11月にはシリーズDで23億ドルを調達し、評価額は293億ドル（約4兆円超）に到達しました。年間経常収益（ARR）は10億ドルを超え、SaaS史上最速クラスの成長とも評されています。創業者4名全員がビリオネアとなったことでも話題を呼びました。

VS Code Forkの経緯と意義

なぜCursorはVS Codeをフォークしたのでしょうか。その理由は「AIをエディタの拡張機能ではなく、エディタ体験そのものに統合する」というビジョンにあります。VS Codeは世界で最も使われているコードエディタであり、その拡張性と豊富なエコシステムは比類なき強みです。しかし、拡張機能APIの制約から、AIによるインライン編集の提案、マルチファイルの同時編集、タブ補完の高度なカーソル予測といった機能を拡張機能だけで実現するのは困難でした。

Cursorはこの問題をフォークによって解決し、エディタのレンダリングエンジンやエディタコアに直接手を加えることで、AIのコード提案をシームレスに表示・適用できるようにしました。同時に、VS Codeのコアリリースを定期的にマージ（通常は2回に1回のメインラインリリース）しており、VS Codeの安定性とセキュリティパッチの恩恵も受けることができます。VS Codeの拡張機能もそのまま使えるため、既存のワークフローをほぼ維持したまま移行できる点もユーザーにとっての大きなメリットです。

ポイント: CursorはVS Codeの「上位互換」であり、VS Codeユーザーは既存の設定・拡張機能・キーバインドをワンクリックでインポートできます。VS Codeの使い方を知っていれば、Cursorの基本操作は即日マスターできます。

Section 02

主な機能 — Tab補完・Chat・Agent・Cmd+K

Cursorの5大AI機能を詳しく解説。コードを書くすべての瞬間にAIが寄り添います。

⇥

Tab補完（Cursor Tab）

最頻使用機能

Cursor Tabは「ステロイドを打ったオートコンプリート」と呼ばれる、Cursorの核心機能です。従来のIntelliSenseのような単純なトークン補完ではなく、直近の編集履歴、周辺のコードコンテキスト、Linterエラー、さらにはカーソル位置の意図まで読み取り、複数行にわたるコード補完をTabキー一発で適用できます。

特筆すべきは「次の編集位置の予測」機能です。Tabで補完を確定した後、次にカーソルが移動すべき場所をAIが予測し、再度Tabを押すだけで次の編集箇所へジャンプします。これにより、複数箇所の関連する修正を連続的に行えます。タイプ中にリアルタイムでゴースト表示されるサジェストは、Tabで採用、Escで却下でき、使えば使うほど開発者の意図を学習して精度が向上していきます。Cursor Tab専用の軽量モデルが使われており、キーストロークごとにリクエストが送信されるため応答速度は極めて高速です。

💬

Chat（AIチャット）

ショートカット: Cmd+L

サイドパネルで開くAIチャットは、コードに関する質問、説明、デバッグ支援など汎用的な対話に使えます。現在開いているファイルやカーソル位置のコードが自動的にコンテキストとして含まれるため、「この関数は何をしている？」「このエラーの原因は？」といった質問にコードベース固有の文脈で回答してくれます。

@記号を使ったコンテキスト参照も強力で、@fileで特定ファイル、@folderでフォルダ、@webでWeb検索、@docsでドキュメント、@codebaseでコードベース全体の意味検索を指示できます。チャットで提案されたコードは「Apply」ボタンひとつでエディタに反映可能です。

🤖

Agent Mode（エージェントモード）

Cursor 2.0の中核

Agentは、複雑なコーディングタスクを自律的に遂行するCursorの最も強力な機能です。Cmd+Iでサイドペインから起動でき、エージェントが自動でコードベースを探索し、複数ファイルを同時に編集し、ターミナルコマンドを実行し、Linterエラーを検知して修正するという一連の流れを自動で行います。

Cursor 2.0ではマルチエージェント機能が導入され、複数のエージェントを並列で動かして異なるタスクを同時進行させることが可能になりました。さらにBackground Agent機能により、開発者がエディタを閉じている間もクラウド上でエージェントが作業を続け、完了後にPull Requestとして結果を受け取れます。

エージェントの動作にはAgent Mode、Ask Mode、Manual Modeなど複数のモードがあり、Agent Modeがデフォルトです。Agent Modeでは自律的にファイルを探索・編集し、Ask Modeでは読み取り専用で質問に回答し、Manual Modeでは明示的に指示されたファイルのみを編集します。さらにDebug Modeでは、エラーが発生したファイルやスタックトレースに基づいてデバッグを自動化します。

🎵

Composerモデル（独自AIモデル）

Cursor 2.0新機能

Cursor 2.0で発表されたComposerモデルは、Anysphereが独自に開発したソフトウェアエンジニアリング特化のAIモデルです。強化学習（RL）を用いて実際のコードベース内でトレーニングされたMixture-of-Experts（MoE）アーキテクチャで、同等の知能を持つモデルと比較して4倍高速に動作し、ほとんどのターンを30秒以内に完了させます。

Auto選択時にCursorのAgentモードで使用される場合は無制限で利用でき、フロンティアモデルの使用量を節約できるため、コスト効率にも優れています。Composerは低レイテンシーでのエージェントコーディングに最適化されており、マルチステップのコーディングタスクにおいて高い信頼性を発揮します。

✏️

Cmd+K（インライン編集）

ショートカット: Cmd+K / Ctrl+K

Cmd+Kはファイル内でのピンポイントなコード生成・編集のための機能です。コードを選択した状態でCmd+Kを押すと、インラインのプロンプトバーが表示され、「この関数をTypeScriptに変換して」「エラーハンドリングを追加して」といった自然言語の指示でコードを直接編集できます。

選択なしでCmd+Kを使えば、カーソル位置に新しいコードを生成します。また、ターミナル内でCmd+Kを使うと、自然言語からターミナルコマンドを生成することもできます。例えば「ポート3000を使っているプロセスを終了して」と入力すれば、適切なコマンド（lsof -ti:3000 | xargs killなど）を提案してくれます。Cmd+Shift+Enterでファイル全体の編集も可能になりました。

Section 03

アーキテクチャ — VS Code fork、独自拡張、モデル切替

Cursorがどのように構築され、AIリクエストがどう処理されるかを技術的に解説します。

VS Code Fork構成

CursorはMicrosoftがオープンソースとして公開しているVS Code（microsoft/vscodeリポジトリ）をフォークして構築されています。VS CodeのElectronベースのデスクトップアプリケーション構造をそのまま活用しつつ、エディタコア（Monaco Editor部分）やサイドパネル、ステータスバーなどのUI層にAI機能を直接組み込んでいます。VS Codeの2回に1回のメインラインリリースのタイミングでアップストリームをマージしており、セキュリティパッチについてはhigh-severityなものを即座にcherry-pickして独立リリースする体制をとっています。

AI機能のアーキテクチャ

Cursorの全AI機能は、クライアント（Electronアプリ）からAnysphereのAWS上のバックエンドサーバーにリクエストを送信する形で動作します。サーバー側でプロンプト構築が行われた後、適切なモデルプロバイダー（OpenAI、Anthropic、Google、xAIなど）にリクエストがルーティングされます。カスタムモデル（Tab補完用モデルやComposerモデル）はFireworks AI、Baseten、Togetherなどの推論プロバイダーでホスティングされています。

重要なのは、ユーザーが独自のAPIキーを設定した場合でも、リクエストは必ずAnysphereのAWSインフラを経由するという点です。これはプロンプト構築やカスタムモデルの利用がサーバーサイドで行われるためです。現時点ではセルフホスト型のサーバーデプロイオプションは提供されていません。

コードベースインデクシング

Cursorにはセマンティックインデクシング機能があり、プロジェクト内の全コードをベクトル化してインデックスします。仕組みとしては、ファイルのMerkleツリーを計算してハッシュの差分のみをサーバーにアップロードし、サーバー側でチャンク化・エンベディングを行います。エンベディングはTurbopuffer（Google Cloud上のベクトルDB）に保存されますが、Privacy Modeユーザーの場合はプレーンテキストコードはサーバー上に一切保存されません。ファイルパスは暗号化（難読化）された形で保存されます。10分ごとに差分チェックが行われ、変更のあったファイルのみ再インデックスされます。

モデル切替の仕組み

Cursorでは、Chatやエージェントで使用するAIモデルをドロップダウンから簡単に切り替えられます。「Auto」を選択すると、Cursorが容量とタスクの性質に基づいて最適なフロンティアモデルを自動選択します。Auto使用時は無制限にリクエストが送れるため（Proプラン以上）、コスト効率を優先する場合はAutoが推奨されます。特定のモデル（例: Claude Sonnet 4、GPT-4o）を指定した場合は、月次の使用量クレジットを消費します。

VS Codeからの移行手順

Cursorのダウンロードとインストール

cursor.comからmacOS / Windows / Linux向けのインストーラーをダウンロードし、通常のアプリケーションとしてインストールします。VS Codeと並行して使用可能です。

VS Code設定のインポート

初回起動時に「VS Code Import」を促すダイアログが表示されます。あるいは、設定画面の「VS Code Import」セクションから「Import」ボタンをクリック。拡張機能、テーマ、設定、キーバインドが一括で移行されます。

アカウント作成・サインイン

GitHubまたはGoogleアカウントでサインインします。Hobbyプランは無料で、2週間のPro体験版も付属します。

AI機能の動作確認

適当なプロジェクトを開き、コードを書いてTab補完が動くか確認します。Cmd+Lでチャット、Cmd+Iでエージェントを試してみましょう。

（任意）Privacy Modeの設定

企業利用やプライベートプロジェクトの場合は、Settings → Cursor Settings → Privacy ModeをONにします。コードが学習データに使われない保証が得られます。

移行のハードル: 「移行は驚くほど簡単だった。全ての拡張機能がそのまま動作した」という声が多数。VS Codeのキーバインドもそのまま使えるため、操作感の違和感はほぼありません。唯一、拡張機能の署名検証がデフォルトOFFなので、セキュリティに敏感な場合はextensions.verifySignatureをtrueに設定することを推奨します。
Section 07

.cursorrules・プロジェクト設定のベストプラクティス

AIの出力品質を劇的に向上させるプロジェクト固有のルール設定方法

Cursorの出力品質を最大化する最も効果的な方法のひとつが、Cursor Rulesの活用です。ルールはAIにプロジェクト固有の文脈—使用技術、コーディング規約、アーキテクチャ方針—を伝え、一貫した高品質なコード生成を可能にします。

ルールの種類と配置

Cursor v0.45以降、ルールシステムが大幅に刷新されました。従来のプロジェクトルート直下に.cursorrulesファイルを置く方式に加え、.cursor/rules/ディレクトリ内に複数のルールファイルを配置する新方式が導入されています。新方式では用途ごとにルールを分割し、それぞれに適用条件（グロブパターンなど）を設定できます。

ルールには大きく3つのスコープがあります。Project Rulesは.cursor/rules/に配置しGit管理してチームで共有するルール、User RulesはSettings画面で個人設定するグローバルルール（全プロジェクト共通）、Auto-attached Rulesはファイルパターンに応じて自動適用されるルールです。

# プロジェクト構造の例 my-project/ ├── .cursor/ │ └── rules/ │ ├── general.mdc # 全体的なコーディング規約 │ ├── react.mdc # React特有のルール (*.tsx に自動適用) │ ├── api.mdc # API開発ルール (src/api/** に自動適用) │ └── testing.mdc # テストルール (*.test.* に自動適用) ├── .cursorignore # AIに送信しないファイルを指定 ├── src/ └── ...
# .cursor/rules/general.mdc の例 # description: プロジェクト全体の規約 # globs: **/* ## プロジェクト概要 これはNext.js 15 + TypeScript + Tailwind CSSで構築されたWebアプリケーションです。 ## コーディング規約 - TypeScriptのstrictモードを使用する - 関数コンポーネントとReact Hooksを使用する（クラスコンポーネント禁止） - 変数名・関数名はcamelCase、コンポーネント名はPascalCase - CSSはTailwind CSSのユーティリティクラスで記述（カスタムCSS最小限） - エラーハンドリングは必ずtry-catchで行い、ユーザーへの適切なフィードバックを含める ## アーキテクチャ - App Routerを使用（pages/は使わない） - Server Componentsをデフォルトとし、必要な場合のみ'use client'を付与 - データフェッチにはServer Actionsを優先使用 ## 禁止事項 - anyの使用禁止（必ず具体的な型を定義する） - console.logのcommit禁止（デバッグにはloggerユーティリティを使用） - マジックナンバー禁止（定数として定義する）

ベストプラクティス一覧

🎯 焦点を絞る

ルールは短く、具体的で、実行可能なものにします。あまりに多くのルールを詰め込むとモデルのパフォーマンスが低下します。1ファイルあたり500行以内を目安に。

📂 関心ごとに分離する

フロントエンド、バックエンド、テストなど、関心領域ごとにルールファイルを分け、globsで適用範囲を限定します。全ファイルに適用するルールは最小限に。

✅ 正例と負例を示す

「こう書いてほしい」「こう書いてほしくない」の両方の具体例をコードで示すと、AIの理解が格段に向上します。

🔄 .cursorignoreを活用する

node_modules、.env、機密ファイル、大規模バイナリなどを.cursorignoreに追加し、AIに送信されるコンテキストを整理します。.gitignoreのフォーマットと同じです。

📖 PRDを活用する

プロジェクトの要件定義書（PRD）を.cursor/rules/やプロジェクトルートに配置しておくと、AIがプロダクトの文脈を理解した上でコードを生成します。

🔧 Gitで管理・共有する

.cursor/rules/はGitに含めてチーム全体で共有します。コードレビューの際にルールの改善も議論しましょう。

参考リソース: awesome-cursorrules（GitHub）には各フレームワーク・言語向けのルールテンプレートが多数公開されています。またcursorrules.orgにもガイドと例があります。
Section 08

セキュリティ・プライバシー

Privacy Mode、SOC 2認証、データ処理の仕組みを詳しく解説します。

Privacy Modeの保証

CursorのPrivacy Modeは、コードデータがモデルプロバイダーに保存されたり、学習に使用されたりしないことを保証する機能です。設定画面またはチーム管理者がONにでき、全プランで利用可能（Hobbyプランでも可）。チームプランの場合はデフォルトで強制有効化されます。Cursorユーザーの50%以上がPrivacy Modeを有効にしているとのことです。

Privacy Modeの実装は技術的にも堅牢です。各リクエストにはx-ghost-modeヘッダーが含まれ、バックエンドのプロキシがこのヘッダーに基づいてリクエストをPrivacy Mode専用レプリカとそれ以外のレプリカに振り分けます。Privacy Mode用レプリカではすべてのログ関数がデフォルトでno-op（何もしない）になっており、コードデータが誤ってログに記録されることを防止しています。バックグラウンドタスク用のキューとワーカーも同様に並列化されています。

ヘッダーが欠落した場合のデフォルトはPrivacy Mode ON。チーム強制設定のチェックはクライアント側で5分ごと、サーバー側でもホットパスで行われ、キャッシュミス時もPrivacy Mode前提で処理されます。つまり、チームに参加してから最大5分以内にPrivacy Modeが保証される設計です。

SOC 2 Type II認証とセキュリティ体制

CursorはSOC 2 Type II認証を取得済みで、詳細なレポートはtrust.cursor.comからリクエストできます。また、定期的な（少なくとも年1回の）第三者ペネトレーションテストを実施しています。インフラストラクチャのアクセスは最小権限原則に基づき割り当てられ、AWSへのアクセスには多要素認証が強制されています。

データフローとサブプロセッサー

CursorのAIリクエストは必ずAWS上のAnysphereサーバーを経由します。主要インフラはAWS（US）に配置され、一部のレイテンシー重要サービスがEUとシンガポールにあります。コードデータを参照するサブプロセッサーには、Cloudflare（リバースプロキシ）、Azure・GCP（補助インフラ）、Fireworks・Baseten・Together（カスタムモデル推論）、OpenAI・Anthropic・Google Vertex・xAI（フロンティアモデル推論）があり、全プロバイダーとZero Data Retention契約を締結しています（Privacy Modeユーザー対象）。中国にインフラはなく、中国企業をサブプロセッサーとしても使用していないとのことです。

🔒 .cursorignoreファイル

特定のファイルやディレクトリをCursorのサーバーに送信させないためのファイルです。.gitignoreと同じ構文で記述します。機密情報（.env、秘密鍵、顧客データなど）や送信不要なファイル（バイナリ、ビルド成果物など）を指定しましょう。

# .cursorignore の例 .env* *.pem *.key secrets/ customer-data/ *.sqlite dist/ build/

⚠️ セキュリティ上の注意点

Workspace TrustがデフォルトOFF — 信頼できないフォルダを開く際は手動で有効化を検討してください。拡張機能の署名検証もデフォルトOFF — セキュリティ重視の場合はextensions.verifySignature: trueを設定してください（ただし現時点では毎回警告が表示されます）。また、エンベディングの逆変換による情報漏洩リスクは完全にはゼロではありませんが、現実的な攻撃は困難とされています。

アカウント削除: 設定ダッシュボードからいつでもアカウント削除が可能で、30日以内にすべてのデータ（インデックス済みコードベース含む）が完全に削除されます。ただし、Privacy ModeがOFFの期間中にデータが学習に使用された場合、既存の訓練済みモデルは即座に再訓練されませんが、将来のモデル訓練からは除外されます。

ユースケース集（10選）

Cursorが特に威力を発揮する10の実践的なシナリオを紹介します。

01

新規プロジェクトのスキャフォールディング

「Next.js 15 + TypeScript + Tailwind CSSで認証付きSaaSのボイラープレートを作って」とAgentに指示するだけで、ディレクトリ構造、設定ファイル、基本コンポーネントが一式生成されます。PRDファイルをコンテキストに含めれば、プロダクト要件に沿った構造で出力してくれます。

02

レガシーコードのリファクタリング

古いJavaScript（ES5）やjQueryベースのコードをモダンなReactやTypeScriptに変換。Agentが複数ファイルを横断して依存関係を追跡しながらリファクタリングします。型定義の自動生成やテストの追加も同時に指示可能。

03

バグの調査と修正

エラーメッセージやスタックトレースをチャットに貼り付けると、原因の特定から修正コードの提案まで一気に行います。Debug Modeではエラー発生ファイルに基づいた自動修正も。Linterエラーの自動修正もTabでそのまま適用できます。

04

テストコードの自動生成

既存の関数やコンポーネントを選択し、Cmd+KまたはAgentで「単体テストを書いて」と指示。Jest、Vitest、Pytestなどのフレームワークに合わせたテストコードが生成されます。カバレッジの向上や境界値テストの追加にも対応。

05

API設計とバックエンド実装

OpenAPIスキーマやPrismaモデルをコンテキストに含め、RESTful APIやGraphQLリゾルバーを一括生成。バリデーション、エラーハンドリング、認証ミドルウェアも含めた実用的なコードが得られます。

06

UIコンポーネントの高速プロトタイピング

デザインの説明（またはスクリーンショットをGPT-4oに送信）からReact/Vue/Svelteコンポーネントを生成。Tailwind CSSやshadcn/uiを使ったスタイリング指示も自然言語で可能。内蔵ブラウザで即座にプレビュー。

07

ドキュメント・コメントの自動生成

既存コードからJSDoc、docstring、READMEを自動生成。@codebaseを使えばプロジェクト全体を理解した上で、アーキテクチャ概要やAPI仕様書も作成してくれます。

08

コードレビューの補助

差分をチャットに投げ込み「この変更をレビューして」と依頼すると、潜在的なバグ、パフォーマンス問題、セキュリティリスクを指摘。BugBotを使えばPRの自動レビューも可能です。

09

新技術・ライブラリの学習

@docsでライブラリの公式ドキュメントを参照しつつ、「このライブラリの使い方をサンプルコード付きで教えて」と質問。未知のフレームワークやAPIを使ったコードも、文脈に沿った正確なサンプルが得られます。

10

データベースマイグレーション・スキーマ変更

既存のスキーマ定義を読み取り、マイグレーションファイルの生成、シードデータの作成、関連するモデルやリポジトリコードの更新を一括で処理。Prisma、TypeORM、Drizzleなど主要ORMに対応。

Section 11

始め方ガイド — ゼロからCursorを使い始める

ダウンロードから最初のAIコーディング体験までを丁寧にガイドします。

Cursorのダウンロードとインストール

cursor.comにアクセスし、お使いのOS（macOS / Windows / Linux）に対応したインストーラーをダウンロードします。macOSの場合は.dmgをダウンロードしてApplicationsフォルダにドラッグ、Windowsは.exeを実行、Linuxは.AppImageまたは.debパッケージを利用します。インストール後に起動すれば初回セットアップウィザードが始まります。

初回セットアップ — テーマとキーバインド

初回起動時にエディタのテーマ（Dark/Light）とキーバインドスタイルの選択画面が表示されます。VS Codeから移行する場合は「VS Code」キーバインドを選択します。Vim、Emacs、Sublime Textなど他のエディタのキーバインドも選べます。

VS Codeの設定をインポート（オプション）

VS Codeを使っていた場合、「Import VS Code Settings」を選択すると拡張機能、テーマ、設定、キーバインドがすべてインポートされます。後からでも Settings → VS Code Import から実行できます。

アカウント作成とサインイン

GitHub、Google、またはメールアドレスでアカウントを作成します。Hobbyプランは無料で、クレジットカードの登録は不要です。新規ユーザーには2週間のPro体験版が付属するので、全機能を試すことができます。

プロジェクトを開いて最初のAI体験

File → Open Folder でプロジェクトフォルダを開きます（またはCmd+O）。コードベースのインデクシングが自動で開始されます（Settings → Indexing で状況を確認可能）。任意のファイルを開いてコードを書き始めると、Tab補完が自動的に提案を表示します。Tabで採用、Escで却下です。

チャットを使ってみる

Cmd+L（Mac）またはCtrl+L（Windows/Linux）でサイドパネルのチャットが開きます。「このファイルの概要を説明して」「ここにエラーハンドリングを追加するにはどうすればいい？」などと質問してみましょう。コードの選択状態やカーソル位置が自動的にコンテキストに含まれます。

Agentモードを使ってみる

Cmd+I（Mac）でエージェントパネルを開き、「ログイン機能を実装して」「READMEを作成して」などタスクを指示します。エージェントがコードベースを探索し、必要なファイルを作成・編集し、ターミナルコマンドも実行します。提案された変更はdiff表示されるので、「Accept」で採用します。

Cmd+Kでインライン編集を試す

コードを選択してCmd+Kを押し、「TypeScriptに変換して」「この関数をasync/awaitに書き換えて」などと入力してみましょう。選択範囲が直接編集されます。ターミナル内でCmd+Kを使うと、自然言語からシェルコマンドが生成されます。

.cursorrules を設定する

プロジェクトに合わせた.cursor/rules/ディレクトリを作成し、コーディング規約や技術スタックの情報を記述します。これによりAIの出力品質が劇的に向上します。まずは簡単なルールから始めて、使いながら洗練していきましょう。

より深く使いこなす

@codebaseでプロジェクト全体に質問、@webでWeb検索、@docsでドキュメント参照、@fileで特定ファイルを明示参照、といった@コマンドを活用しましょう。モデルの切り替えも試して、タスクに最適なモデルを見つけていきます。
