01

Windsurfとは — 概要・経緯・買収の背景

Windsurfは、AI技術を中核に据えた「エージェンティックIDE（Agentic IDE）」です。単なるコード補完ツールではなく、AIがコードベース全体を理解し、開発者と協働しながら複数ステップのタスクを自律的に遂行できる、次世代の統合開発環境として設計されています。2024年11月に「世界初のエージェンティックIDE」として発表され、わずか数ヶ月で100万人以上の開発者に利用されるまでに急成長しました。

Windsurfの最大の特徴は、「Flows」と呼ばれる概念にあります。これは従来のAIアシスタント（コパイロット型）と自律型エージェントの長所を融合したもので、AIが開発者の意図やリアルタイムの操作を理解しながら、文脈に沿った提案・コード編集・コマンド実行を行います。開発者は常にループの中にいながらも、反復的な作業をAIに任せることができるため、「風に乗ってコーディングする」ような滑らかな体験が得られます。これがWindsurf（ウィンドサーフ）という名前の由来でもあります。

2026年3月現在、WindsurfはCognition（自律型AIソフトウェアエンジニア「Devin」の開発元）の傘下にあり、独立したプロダクトとして運営を継続しています。Cognitionは Windsurf の IP、プロダクト、商標、ブランド、そしてユーザーベースを取得し、Devin と Windsurf の二本柱でAIコーディング領域をリードする構えです。

02

主な機能 — Flows・Cascade・AI補完・エージェントモード

Windsurfは単なるコードエディタではなく、AIを最深部に統合した「AIネイティブIDE」です。その機能は大きく分けて、パッシブ体験（自動補完系）、インストラクティブ体験（チャット・コマンド系）、エージェンティック体験（Cascade系）の3層で構成されています。それぞれが連携し、開発者がフロー状態を維持できるよう設計されています。

🌊

Flows（フロー）

WindsurfのAI哲学の中核概念。コパイロット型（人間主導）とエージェント型（AI主導）のハイブリッドアプローチ。AIが開発者のリアルタイム操作（エディット、ターミナル操作、クリップボード、ブラウザ操作）を認識し、文脈に応じて適切な介入度合いを自動調整します。開発者は指示を出すだけで、AIが適切な粒度で作業を引き受けます。

⚡

Cascade（カスケード）

Windsurfの中核AIエージェント。Code モードとChat モードの2つの動作モードを持ちます。Code モードではコードベースの直接編集が可能で、Chat モードではコードに関する質問や一般的なプログラミング原則の解説に最適化されています。マルチステップの推論とツール呼び出しを交互に行う真のエージェント動作を実現します。

📝

Windsurf Tab / Supercomplete

キーストロークごとにAIが次のコードを予測するパッシブな補完機能。Supercompleteは単なる挿入だけでなく、削除や編集の差分（diff）も提案できる高度なモードです。ターミナルの出力やテスト結果からもコンテキストを取得し、より精度の高い補完を実現します。Tab一つでインポート文の追加やカーソルジャンプも可能です。

🧠

Memories（メモリー）

開発者の好み、プロジェクトの慣例、コーディングスタイルなどをAIが学習・記憶する機能。Cascade との会話中に「これを覚えておいて」と指示するか、設定画面から直接登録できます。プロジェクト固有のメモリーとグローバルメモリーを使い分けることで、AIの出力品質が飛躍的に向上します。

📋

Rules（ルール）

プロジェクトに適用するコーディング規約やAIの振る舞いを定義する機能。.windsurfrules ファイルや設定UIから定義可能。「TypeScript では常に strict モードを使用」「コンポーネントは function 宣言で書く」といった指示を永続的に適用できます。ワークスペース単位、グローバル単位での設定が可能です。

🔄

Workflows（ワークフロー）

複数のステップからなる構造化されたタスクシーケンスを定義する機能。ルールがプロンプトレベルの指示であるのに対し、ワークフローはタスク全体のフローを制御します。「新しいAPIエンドポイントを作る→テストを書く→ドキュメントを更新する」のような一連の作業を自動化できます。

Cascade の詳細機能

ツール呼び出し（Tool Calling）

Cascade は Search、Analyze、Web Search、MCP（Model Context Protocol）、ターミナルなど多彩なツールを自在に使い分けます。必要なパッケージの検出・インストールも自動で行います。「プロジェクトをどうやって実行するの？」と聞くだけで、依存関係のインストールからサーバー起動まで自動対応します。

計画とTodoリスト

複雑なタスクでは、専門の計画エージェントがバックグラウンドで長期プランを継続的に改善します。選択したモデルがプランに基づいて短期的なアクションを実行し、Todoリストで進捗を追跡。会話中に計画の変更を指示することも可能です。

チェックポイントとリバート

Cascade が行った変更はいつでもリバート可能。名前付きチェックポイントを作成して、プロジェクトの特定の状態に素早く戻ることができます。実験的な変更を安全に試せる「セーフティネット」として機能します。

リアルタイム認識

Windsurf独自の機能。開発者のリアルタイムの操作（ファイル編集、ターミナル操作、クリップボードの内容）をAIが自動的に認識するため、いちいちコンテキストを説明する必要がありません。「続けて」と指示するだけで、直前の作業の文脈を引き継いで動作します。

音声入力

音声で Cascade に指示を出すことが可能。現在は音声→テキスト変換による入力をサポートしており、ハンズフリーでの操作を実現します。

同時実行 Cascade

複数の Cascade セッションを同時に実行可能。異なるタスクを並列で進められます。同じファイルを編集する可能性がある場合は、Git worktree 機能と組み合わせて安全に隔離できます。

その他の便利機能

Previews（プレビュー）

フロントエンド開発中にブラウザプレビューをIDE内で直接表示。Cascade はプレビューの内容も認識してフィードバックに活用します。変更がリアルタイムで反映されます。

Deploys（デプロイ）

Teams プランでは、管理者が設定した Netlify アカウントへ直接デプロイ可能。開発からデプロイまでをIDE内で完結できます。

Linter 統合

Cascade が生成したコードに対してリントエラーを自動修正。デフォルトで有効化されており、リント修正にはクレジットが消費されません。コード品質を維持しながらAI生成を活用できます。

.codeiumignore

Cascade に見せたくないファイルやディレクトリを .gitignore と同じ形式で指定可能。エンタープライズ向けにはグローバルな .codeiumignore もサポートし、全リポジトリ横断で制御できます。

@メンション機能

過去の会話を @メンションで参照可能。Cascade が過去の会話から最も関連性の高い情報を自動取得し、文脈として活用します。チーム内での会話共有機能もあります。

Fast Context

独自の swe-grep モデルを使用した高速なコンテキスト検索。大規模なモノレポでもミリ秒単位で関連コードを取得し、Cascade の応答品質を大幅に向上させます。

03

アーキテクチャ — VS Code Fork・独自AI統合

Windsurf Editorは、Microsoft が公開しているVisual Studio Code（VS Code）のオープンソースコードベースをフォークして構築されています。そのため、VS Code ユーザーにとっては馴染み深いUIとキーバインドをそのまま利用でき、学習コストが極めて低く抑えられています。拡張機能は Open VSX Registry を通じてインストール可能で、VS Code のエコシステムの恩恵を受けられます。

Windsurf は上流の microsoft/vscode リポジトリを定期的にマージし、セキュリティパッチが公開された場合は即座にチェリーピックして新バージョンをリリースする体制をとっています。そのため、VS Code の最新機能やセキュリティ修正が迅速に反映されます。

AIの統合は単なるプラグインとしてではなく、エディタのコア部分に深く組み込まれています。キーストロークごとのリクエスト処理、リアルタイムのコンテキスト構築、エージェントのツール呼び出しなど、あらゆるレイヤーでAIが動作しています。この「AI ネイティブ」なアーキテクチャが、単なる VS Code + AI プラグインとは根本的に異なる体験を提供する理由です。

コードベースインデックス

Windsurf のインデックスは抽象構文木（AST）ベースで構築されています。ファイル単位やナイーブなチャンクではなく、関数・メソッド・クラスなどのセマンティックなブロック単位でインデックスを生成するため、大規模なエンタープライズコードでも精度の高いコンテキスト検索が可能です。ローカルインデックスではクライアントマシン上でASTを生成し、各チャンクの埋め込みベクトルをサーバーで計算して、ローカルのベクトルストアに格納します。リモートインデックスでは、ローカルに収まりきらない大規模リポジトリや、現在IDEで開いていないリポジトリのコンテキストも活用できます。

デプロイメントオプション

☁️ Cloud

個人プラン、Teams、Enterprise Cloud。AIリクエストはWindsurfが管理するサーバー（GCP）で処理。Teams/Enterprise はデフォルトでゼロデータ保持。US、EU（フランクフルト）、FedRAMP High（AWS GovCloud）から選択可能。

🔄 Hybrid

最も人気のエンタープライズ構成。データ保持が必要な機能は顧客テナント内のDockerコンテナで実行、推論はWindsurfクラウドで処理。Cloudflare Tunnelで安全に接続。コード片がWindsurf側に保持されることはありません。

🏢 Self-hosted

全コンピュートとデータ保持が顧客のGPU対応テナント内で完結。Docker ComposeまたはHelm Chartで展開。AWS Bedrock、Azure OpenAI等の顧客のプライベートLLMエンドポイントに接続可能。ただし、Windsurf EditorやCascadeの一部先端機能は利用不可。

対応モデルとAIエンジン

Windsurfはマルチモデルアーキテクチャを採用しており、用途やタスクに応じて最適なモデルを選択できます。Cascade の入力ボックス下にあるドロップダウンメニューからモデルを切り替えることが可能です。自社開発のSWEモデルファミリーに加え、主要なAI企業のモデルも利用できます。

06

買収劇の全貌と今後の方向性

Windsurf を巡る2025年の買収劇は、AI 業界で最もドラマチックな出来事の一つでした。その全貌を時系列で整理し、今後の方向性を考察します。

OpenAI $3B 買収交渉とその破綻

2025年4月、OpenAI が Windsurf を約30億ドルで買収する交渉に入ったと CNBC が報道しました。これは Windsurf の年間経常収益（ARR）の約30倍という高い評価額で、OpenAI がモデルメーカーから開発者ツールプラットフォームへ進出する戦略的意図が明確でした。しかし、この取引は複数の要因で破綻しました。

第一に、Microsoft との利害対立です。Microsoft は OpenAI に累計 $13B を投資する最大の投資家であると同時に、GitHub Copilot という競合するAIコーディングツールを持っています。Windsurf の買収は Copilot と直接競合することを意味し、Microsoft 側に大きな懸念を生みました。

第二に、Windsurf チーム側の懸念です。コーディングアシスタントが OpenAI の巨大組織の中でどのようにフィットするか、プロダクトの独立性が維持されるかについて、Windsurf の経営陣が懸念を表明したと報じられています。

Google による人材獲得（$2.4B）

OpenAI との交渉が決裂した2025年7月11日、Google が Windsurf の共同創業者兼CEO Varun Mohan と複数の主要エンジニアを引き抜きました。これは「逆アクハイヤー（reverse acquihire）」と呼ばれる手法で、Google は会社自体を買収するのではなく、テクノロジーのライセンスと人材獲得に対して $2.4B を支払いました。Mohan は Google DeepMind に参加し、AIコーディング関連のプロジェクトに従事しています。

Cognition による買収と現在の体制

Google への人材流出からわずか3日後の2025年7月14日、自律型AIエンジニア「Devin」を開発する Cognition AI が Windsurf の買収を発表しました。Cognition は Windsurf の IP、プロダクト、商標、ブランド、ユーザーベース、そして Google に引き抜かれなかった従業員を取得しました。Jeff Wang が新CEOに就任し、Windsurf は独立したプロダクトとして運営を継続しています。

Cognition はその後2025年9月に $400M の資金調達を実施し、Windsurf + Devin の二本柱戦略に強い支持を集めています。Devin が完全自律型のバックグラウンドエージェントとして位置付けられるのに対し、Windsurf は開発者と協働する「コラボレーティブエージェント」として差別化されています。

今後の方向性

Devin との統合

Cognition は Windsurf の独自技術を Cognition の製品群に段階的に統合する方針です。将来的には、Windsurf IDE 内から Devin のタスクを起動したり、Devin が作成したコードを Windsurf で即座にレビュー・編集したりといったシームレスな連携が期待されます。

エンタープライズ展開の加速

Cognizant との戦略的パートナーシップ（2026年1月発表）に見られるように、大企業向けの展開を加速しています。Devin によるバッチ処理 + Windsurf による対話的開発のコンビネーションは、エンタープライズの大規模開発に最適化されています。

SWE モデルの進化

SWE-1 → SWE-1.5 の進化に続き、さらなるモデル改良が進行中と見られます。ソフトウェアエンジニアリング全般に特化したモデルは、汎用モデルに対する明確な差別化ポイントです。

MCP エコシステム

Model Context Protocol（MCP）への対応により、外部ツールとの連携が拡大しています。データベース、API、CI/CD パイプライン等の外部システムとの統合がますます充実する方向です。

⚠️ 注意： 買収劇の余波として、一部の従業員がレイオフやバイアウトの対象となったことが報じられています。また、元CEOの Mohan が Google に移ったことで、長期的な製品ビジョンに変化が生じる可能性があります。ただし、2026年3月現在、Windsurf は活発にアップデートされ続けており、Cognition による投資も継続しています。

セキュリティ・プライバシー

AIコーディングツールにとって、セキュリティとプライバシーは最も重要な懸念事項の一つです。Windsurfは開発者のコードが知的財産であることを深く認識し、業界でも最高水準のセキュリティ体制を構築しています。Fortune 500企業を含む数千社がWindsurfを企業全体で導入している実績は、その信頼性の証といえます。

🔒

SOC 2 Type II 認証

第三者監査によるSOC 2 Type II認証を取得。年次のペネトレーションテストも実施しています（最終: 2025年2月13日）。セキュリティ開発プロセスと運用管理の信頼性が保証されています。

🏛️

FedRAMP High 認定

米国連邦政府機関向けの最も厳格なセキュリティ認定。コードレビュープロセス、MDM、ゼロトラストVPN、OWASP ASVS Level 1 準拠、災害復旧計画など、SOC 2を超える開発実践が求められます。

🏥

HIPAA コンプライアンス

医療機関向けにHIPAA準拠のプラットフォームを維持。大規模な導入に対しては Business Associate Agreement（BAA）も締結可能です。

🚫

ゼロデータ保持（ZDR）

Teams/Enterprise プランではデフォルトでZDRが有効。コードやコード由来のデータがサーバーやサブプロセッサに平文で保存されることは一切ありません。個人プランでも任意でZDRを有効化可能。ZDR有効ユーザーのデータは学習にも使用されません。

🔐

SSO / アクセス制御

Enterprise プランでは SAML ベースの SSO（Microsoft Entra、Okta、Google Workspaces 等）に対応。RBAC（ロールベースアクセス制御）により、ユーザーの権限を細かく管理できます。

📋

帰属・ライセンスフィルタリング

生成されたコードが非寛容ライセンスのコードに類似する場合、自動的に検出してユーザーに表示しません。Jaccard編集距離を用いたファジーマッチングで、ナイーブな文字列マッチングよりも堅牢な検出を実現。Enterprise向けにはIPインデムニティ条項も提供。

データフローの透明性

Windsurfはデータの流れについて極めて詳細な情報公開を行っています。リクエスト時にはコンテキストとして関連コードスニペット、最近のエディタ操作、会話履歴、ユーザー指定のシグナル（ルール、メモリー等）の組み合わせが送信されます。重要なのは、単一のリクエストにコードベース全体が含まれることはないという点です。先読みパーソナライゼーションのための埋め込み計算でも、コードはクライアント側でASTに分割され、個別のスニペットとして送信されます。

全データは TLS で暗号化 されて転送されます。サブプロセッサとしては GCP、Crusoe、Modal、Oracle Cloud、Palantir（FedRAMP）、AWS、OpenAI、Anthropic、Google Vertex、xAI、Fireworks（DeepSeek）を利用しており、主要な推論プロバイダーとはゼロデータ保持契約を締結しています。Enterprise管理者は特定のプロバイダーの利用を組織レベルで無効化できます。

制限事項・注意点

クレジットシステムの制約

Windsurf のクレジットシステムは、使用するモデルによって消費量が異なります。SWE-1.5 や Claude 4 Opus のようなハイエンドモデルを多用すると、月間クレジットがすぐに枯渇する可能性があります。特に Free プランの25クレジットは、本格的な開発にはまず足りません。Pro プランの500クレジットも、大規模プロジェクトで Cascade を多用すると月半ばで消化してしまうことがあります。追加クレジットは $10/250クレジットで購入可能ですが、ヘビーユースの場合はコストが嵩みます。

VS Code 拡張機能の互換性

Windsurf は VS Code のフォークですが、Microsoft の公式 VS Code Marketplace ではなくOpen VSX Registry を使用します。そのため、一部の VS Code 拡張機能（特に Microsoft 独自のもの、例えば Remote - SSH、Live Share の一部バージョンなど）が利用できないか、動作が異なる場合があります。多くの主要な拡張機能は Open VSX にも登録されていますが、事前に確認が必要です。

Self-hosted の機能制限

Enterprise Self-hosted デプロイメントでは、最大限のセキュリティが確保される代わりに、Windsurf Editor 自体や Cascade の一部先端機能が利用できません。主に IDE プラグイン（JetBrains、VS Code プラグイン等）経由でのAutocomplete/Chat機能に限定されます。最先端のエージェンティック体験を求める場合は、Hybrid デプロイが推奨されます。

AI 生成コードの品質

あらゆるAIコーディングツールに共通する注意点ですが、AI が生成するコードは必ずしも正確ではありません。特に複雑なビジネスロジック、パフォーマンスクリティカルなコード、セキュリティに関わるコードについては、人間による慎重なレビューが不可欠です。Windsurf の Cascade は「コラボレーティブエージェント」として設計されており、ファイル編集は即座にコミットされず、明示的な承認が必要ですが、それでも盲目的にAcceptすることは避けるべきです。

組織的な不確実性

2025年の買収劇で CEO と主要エンジニアが Google に移籍し、残りの組織が Cognition に吸収されるという激動を経験しています。新体制下での長期的なプロダクトロードマップにはある程度の不確実性があります。一方で、SWE-1.5 のリリースや Cognizant とのパートナーシップなど、積極的な動きも続いています。選定にあたっては、この組織変動リスクも考慮する必要があります。

その他の注意点

ネットワーク依存

AI機能はクラウド接続が必須（Self-hosted除く）。オフライン環境ではAI支援が受けられません。エディタ自体はオフラインでも動作しますが、Windsurf を使う意義の大半が失われます。

コンテキストウィンドウの限界

非常に大きなファイルや極めて長い会話では、コンテキストウィンドウの制限に達することがあります。Cascade は定期的に会話を要約・チェックポイントしてこの問題を軽減しますが、完全ではありません。

言語サポートの差

Python、TypeScript/JavaScript、Go 等の主要言語では極めて高い品質ですが、マイナー言語やドメイン固有言語（DSL）では補完精度が低下することがあります。

ターミナルコマンドの安全性

Cascade はデフォルトでターミナルコマンドの実行前に承認を求めますが、自動実行モード（Auto-run）を有効にすると、予期しないコマンドが実行されるリスクがあります。Teams/Enterprise ではこのモードは利用不可です。

各ツールの特徴まとめ

Windsurf

エージェンティック IDE の先駆者。Cascade のリアルタイム認識や Flows の概念は独自性が高く、AI と開発者の協働体験が最も洗練されています。SWE-1.5 の高速推論と自社モデル戦略、FedRAMP High 認定によるエンタープライズ対応力が強み。Cognition 傘下で Devin との将来的な統合にも期待が持てます。価格面でも Cursor より $5 安い $15/月と競争力があります。一方、組織変動による不確実性は考慮すべき点です。

Cursor

Windsurf の最大の競合。同じく VS Code フォークのスタンドアロン IDE で、Composer というエージェント機能や独自モデル Composer-1 を持ちます。過去の会話の参照機能や、コードベース横断のコンテキスト理解に強み。$20/月と Windsurf より高めですが、安定した独立企業としての信頼性があります。小〜中規模プロジェクトでの体験が特に評価されています。

GitHub Copilot

Microsoft / GitHub が提供する最大手のAIコーディングアシスタント。VS Code にネイティブ統合され、最も広いIDE対応（VS Code, JetBrains, Visual Studio, Neovim 等）を誇ります。$10/月と最も安価で、GitHub エコシステム（Issues, PRs, Actions）との統合が最大の強み。Agent Mode も追加されましたが、Windsurf や Cursor のエージェント能力にはまだ及ばないとの評価が多いです。既存のワークフローを大きく変えたくない開発者に最適です。

Claude Code

Anthropic が提供するターミナルネイティブのコーディングエージェント。最大のコンテキストウィンドウ、SWE-bench での最高スコア、Claude モデルの高い推論能力が最大の強み。IDE ではなくターミナルで動作するため、既存のエディタ（VS Code, Neovim 等）と併用する形になります。API 従量課金が基本で、予測不能なコストが欠点。Max プラン（$100〜$200/月）で定額利用も可能。本当に困難な問題に取り組む上級開発者に向いています。

OpenAI Codex

OpenAI が提供するコーディングエージェント。ローカル CLI と クラウドベースの非同期タスク実行の両方をサポートする点が特徴。クラウドモードではサンドボックス環境で非同期にタスクを実行し、結果を後から確認できます。オープンソースの CLI ツールもあり、コミュニティの貢献を受けやすい構造です。GPT-5 や codex-1 モデルとの統合が強み。

💡 選択のポイント： エージェンティック IDE を求めるなら Windsurf か Cursor。コストを最小限に抑えたいなら GitHub Copilot。最高の推論能力を求めるなら Claude Code。非同期タスク実行が重要なら OpenAI Codex。エンタープライズのコンプライアンス要件が厳格なら、FedRAMP High を持つ Windsurf が唯一の選択肢です。

始め方ガイド — インストールから初回利用まで

Windsurf は Mac、Windows、Linux に対応しています。VS Code を使ったことがある方なら、ほぼ同じ操作感で即座に利用開始できます。以下のステップで、AIコーディングの世界へ飛び込みましょう。

Windsurf をダウンロード

windsurf.com/editor にアクセスし、お使いの OS に対応するインストーラーをダウンロードします。Mac（Apple Silicon / Intel）、Windows（64bit）、Linux（.deb / .rpm / tar.gz）が用意されています。ダウンロードしたインストーラーを実行するだけでインストールが完了します。

アカウント作成・ログイン

初回起動時にサインインを求められます。メールアドレス、Google アカウント、GitHub アカウントのいずれかでサインアップ可能です。Free プランで開始され、25クレジットが付与されます。

VS Code の設定をインポート（任意）

既存の VS Code ユーザーは、設定、拡張機能、キーバインドをインポートできます。オンボーディングフローで案内されるので、指示に従って操作してください。テーマ、フォント、お気に入りの拡張機能がそのまま使えます。

プロジェクトフォルダを開く

「Open Folder」から既存のプロジェクトを開くか、Cascade を使って新しいプロジェクトをゼロから生成できます。フォルダを開くと、バックグラウンドでローカルインデックスの構築が開始されます。

Cascade を開いてみよう

Cmd+L（Mac）/ Ctrl+L（Windows/Linux）を押すか、右上のCascadeアイコンをクリックします。まずは Chat モードで「このプロジェクトの構造を教えて」と聞いてみましょう。AI がプロジェクトを分析して説明してくれます。

Code モードでコードを書いてもらう

Cascade のモードを「Code」に切り替えて、「ユーザー認証機能を追加して」「このコンポーネントにローディング状態を追加して」のように指示します。AI が計画を立て、ファイルを作成・編集します。変更は自動的には適用されず、「Accept」ボタンで承認する必要があります。

Tab 補完を体験する

通常のコーディング中に、グレーのテキストで補完候補が自動表示されます。Tab キーで受け入れ、Esc でスキップ。Supercomplete モードでは、カーソル周辺に挿入・削除の差分が小さなウィンドウで表示されます。設定から Autocomplete と Supercomplete を切り替えられます。

Rules と Memories を設定する

Cascade 右上のカスタマイゼーションアイコンから、ルールとメモリーを設定します。「TypeScript で strict モードを常に使う」「コメントは日本語で書く」のようなルールを追加すると、AIの出力が一貫性を持つようになります。プロジェクトの .windsurfrules ファイルに記述することもできます。

モデルを選択する

Cascade の入力ボックス下のドロップダウンからモデルを選択します。Free プランでは 0クレジットのモデル（SWE-1系の一部）が無制限に使えます。Pro プランなら SWE-1.5、Claude 4 Sonnet、GPT-4.1 などのプレミアムモデルも利用可能です。タスクの複雑さに応じてモデルを使い分けましょう。

プレビューとデプロイ

フロントエンド開発なら、Previews 機能で IDE 内にブラウザプレビューを表示。変更がリアルタイムで反映されます。Teams プランなら Deploys 機能で Netlify への直接デプロイも可能。開発からデプロイまで IDE を離れる必要がありません。

🔧 既存 IDE ユーザーへ： Windsurf は IDE プラグインとしても利用可能です。JetBrains（IntelliJ, PyCharm, WebStorm 等）、VS Code、Visual Studio、Vim/NeoVim、Jupyter、Chrome でプラグインが提供されています。docs.windsurf.com/plugins からインストールできますが、Cascade のフルエージェント機能は Windsurf Editor でのみ利用可能です。

生産性を最大化する10のTips

1. ルールを先に書く

プロジェクト開始時に .windsurfrules を作成し、使用言語、フレームワーク、コーディング規約、ディレクトリ構造の慣例を記述しておくと、AIの出力品質が格段に向上します。

2. メモリーを活用する

「このプロジェクトでは Supabase を使っている」「APIのベースURLは /api/v2」のような情報をメモリーに登録。毎回説明する手間が省け、AIの応答が正確になります。

3. チェックポイントをこまめに作る

大きな変更の前には名前付きチェックポイントを作成。実験的な変更を気軽に試せて、うまくいかなければ一瞬で元に戻せます。

4. @メンションで文脈を引き継ぐ

過去の会話で解決した内容を新しい会話で参照したい時は @mention を使用。Cascade が過去の文脈から最も関連性の高い情報を自動抽出します。

5. モデルを使い分ける

簡単な質問やコード補完には SWE-1.5（高速・低コスト）、複雑な設計やリファクタリングには Claude 4 Sonnet/Opus（高精度）を使い分けることで、クレジットを効率的に消費できます。

6. Queued Messages を活用

Cascade が作業中でも次のメッセージをキューに入れられます。「完了したら次はテストを書いて」のように先回りして指示を出すと、待ち時間が減少します。

7. .codeiumignore を設定する

node_modules、ビルド出力、大きなデータファイルなど、AIに見せる必要のないファイルを除外。インデックスの精度向上とノイズ削減に効果的です。

8. 同時 Cascade を活用する

フロントエンドとバックエンドの作業を別々の Cascade で並行して進められます。Git worktree と組み合わせると、ファイル競合も回避できます。

9. Web Search を有効活用

最新のライブラリドキュメントや API 仕様が必要な場合、Cascade の Web Search 機能を使えば、知識のカットオフを超えた最新情報を取得してコードに反映できます。

10. Workflows で反復作業を自動化

よくある一連の作業（コンポーネント作成→テスト作成→Storybook記述など）をワークフローとして定義。次回から同じ構造のタスクをワンコマンドで実行できます。
