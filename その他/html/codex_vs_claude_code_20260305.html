    Chapter 01 — Overview & History

    両者の概要と歴史的経緯

    OpenAI CodexとAnthropic Claude Codeは、ともにAIを活用したコーディングエージェントとして、2025年から2026年にかけて開発者の日常ワークフローに不可欠なツールとなった。しかし、その誕生の背景と進化の軌跡は大きく異なる。両者の歴史を辿ることで、現在の設計思想の違いが浮き彫りになる。

    OpenAI Codexの歴史

    OpenAI Codexの起源は2021年に遡る。当初はGPT-3をベースに、GitHubの公開リポジトリで学習されたコード生成専用モデルとしてAPIで提供された。これはGitHub Copilotの基盤技術となり、AI支援コーディングの商業的成功を最初に証明したモデルであった。しかし2023年、OpenAIはCodex APIのアクセスを一時停止し、研究者からの反発を受けて学術用途のみ復活させた。2024年はCodexとして表立った動きは少なく、OpenAIのフォーカスはGPT-4、o1モデル系列へと移っていた。

    大きな転換点は2025年4月、Codex CLIがオープンソースとして公開されたことである。TypeScriptで書かれた軽量なターミナルエージェントで、o3やo4-miniモデルを活用してローカル環境で動作した。続いて2025年5月16日、クラウドベースのCodexエージェントがcodex-1（o3の最適化版）と共に発表された。各タスクが独立したクラウドサンドボックスで実行される非同期並列ワークフローが最大の特徴であった。

    2025年後半、Codex CLIはTypeScriptからRustに完全書き直しされ、ゼロ依存のネイティブバイナリとなった。2025年12月にはGPT-5.2-Codexが登場し、コンテキストコンパクションや大規模コード変更の性能が向上。そして2026年2月5日、GPT-5.3-Codexが発表された。これはOpenAI史上初めて「自身の構築に関与した」モデルであり、自身のトレーニングのデバッグやデプロイメント管理を支援したという画期的な存在である。同日、macOS向けCodexデスクトップアプリが発表され、マルチエージェント管理のための統合インターフェースが提供された。

    Anthropic Claude Codeの歴史

    Anthropic Claude Codeの歴史はより新しい。Anthropic自体は2021年にOpenAIの元メンバーによって設立され、AIの安全性に重点を置いたアプローチで知られる。Claude（初代）は2023年に一般公開され、Claude 2、Claude 3（Haiku/Sonnet/Opus）と進化。コーディング能力は世代を重ねるごとに向上していた。

    Claude Codeはコマンドラインツールとして2025年初頭に登場した。ターミナルから直接プロジェクトを操作できるエージェントとして設計され、ファイルの読み書き、コマンド実行、Git操作など包括的な開発支援を提供した。2025年5月時点でベータ版として公開され、VS Code拡張やJetBrainsプラグインも同時期にリリースされた。

    2025年9月にはVS Codeネイティブ拡張のアップデートとチェックポイント機能が追加され、自律的な開発をより安全に行える仕組みが整った。2025年11月にはMCP（Model Context Protocol）による高度なツール統合が公式発表された。12月にはSlack統合がリリースされ、チャットスレッドから直接コーディングタスクを起動できるようになった。

    2026年2月にはClaude Opus 4.6が発表され、コンテキストウィンドウが最大1Mトークン（ベータ）に拡大。同時にAgent Teams（研究プレビュー）が実験的に導入され、複数のClaudeインスタンスが協調して作業するマルチエージェントオーケストレーションが可能になった。2026年2月28日時点でGitHubスター71,500、VS Code拡張のインストール数5.2M、1日あたり約135KのGitHubコミット（全公開コミットの約4%）を記録している。

    Chapter 02 — Design Philosophy

    設計思想の根本的違い

    OpenAI CodexとClaude Codeは、AIコーディングエージェントという同じカテゴリに属しながら、その設計思想において根本的に異なるアプローチを採用している。この違いは、両社の企業DNAそのものに由来する。

    OpenAI Codex：非同期委任（Async Delegation）

    OpenAIのCodexは「開発者が仕事を委任し、結果を確認する」という非同期ワークフローを中核に据えている。Codexのクラウドモードでは、各タスクが独立したサンドボックスコンテナで実行され、開発者はタスクを投げた後、他の作業を行い、完了したら差分をレビューする。これは「同僚への仕事の依頼」に近い体験を目指したものであり、OpenAIは「非同期マルチエージェントワークフローが、エンジニアが高品質なコードを生産するデファクト標準になる」と明言している。

    この設計思想の背景には、OpenAIの「AGI（汎用人工知能）への最短経路」というミッションがある。Codexはコーディング支援を超え、GPT-5.3-Codexでは「コンピュータ上でのあらゆる作業を遂行できるエージェント」への進化を志向している。OSWorldベンチマーク（デスクトップ環境での生産性タスク）で64.7%を達成し、プレゼンテーション作成やスプレッドシート分析など、非コーディングタスクにも対応範囲を拡大している。

    Anthropic Claude Code：開発者協調（Developer-in-the-Loop）

    一方、Claude Codeは「開発者が常にループの中にいる」ことを重視する設計思想を持つ。ローカルマシンで動作し、開発者のプロジェクトディレクトリ内で直接ファイルを操作する。変更はリアルタイムで確認でき、承認を求めるプロンプトにより、エージェントの行動を開発者が常に監督できる。

    Anthropicの企業DNAはAI安全性研究にあり、その思想はClaude Codeのあらゆる側面に反映されている。CLAUDE.mdによるプロジェクト固有の指示、Hooksシステムによる決定論的制御、権限の粒度の細かい管理など、「AIが何をしているかを開発者が常に把握・制御できる」仕組みが随所に組み込まれている。

    本質的な違い：分離 vs 統合

    両者の設計思想の本質的な違いは「分離（Isolation）」と「統合（Integration）」というキーワードに集約される。Codexはタスクをクラウドサンドボックスに分離し、安全性と並列性を確保する。Claude Codeは開発者の既存環境に統合し、文脈の深さとカスタマイズ性を確保する。Codexは「結果」を重視し、Claude Codeは「過程」を重視する。Codexはスループット最適化、Claude Codeは品質と予測可能性の最適化と言い換えることもできる。

    ⚡
    Codexの思想：「委任と並列」
    タスクを投げて放置し、結果をレビューする。複数タスクを同時実行。「同僚に仕事を頼む」感覚。クラウドサンドボックスによるセキュリティ分離。AGIに向けた汎用エージェントへの進化を志向。
    ◈
    Claude Codeの思想：「協調と制御」
    開発者がループの中で対話しながら進める。ローカル実行で完全な文脈を保持。CLAUDE.md、Hooks、権限管理による細やかな制御。AI安全性を企業DNAとした信頼性重視の設計。

    Chapter 04 — Foundation Models

    基盤モデルの違い

    AIコーディングエージェントの能力は、その基盤となるLLMの性能に直結する。OpenAIとAnthropicは異なるモデルアーキテクチャと学習手法で競争しており、2026年3月時点のラインナップは以下の通りである。

    OpenAI Codexのモデル系列

    Codexの最上位モデルはGPT-5.3-Codex（2026年2月5日リリース）である。これはGPT-5.2-Codexのフロンティアコーディング性能とGPT-5.2の推論・専門知識能力を統合し、さらに25%高速化したモデルである。SWE-Bench Pro（公開）で56.8%、Terminal-Bench 2.0で77.3%、OSWorld-Verifiedで64.7%を達成。コンテキストウィンドウは400Kトークン（272K入力 + 128K出力）。APIの価格は入力$1.75/1Mトークン、出力$14.00/1Mトークンとされる。

    日常的なタスク向けにはGPT-5.1-Codex-Miniがあり、レイテンシを抑えたQ&Aやコード編集に最適化されている。入力$1.50/1M、出力$6/1Mで、使用制限を約4倍に延長できる。さらにChatGPT Pro向けには研究プレビューとしてGPT-5.3-Codex-Sparkが提供されており、Cerebras WSE-3上で1,000+ tokens/secという驚異的な速度を実現している。

    Codexモデルの特徴は、強化学習（RL）で実際のコーディングタスクに最適化されていることである。人間のコーディングスタイルやPRの好みに合わせたクリーンなパッチを生成し、テストが通るまで反復的に実行する能力を持つ。

    Anthropic Claude Codeのモデル系列

    Claude Codeの最上位モデルはClaude Opus 4.6（2026年2月リリース）である。SWE-bench Verifiedで80.8%、SWE-bench Proで59%（WarpGrep v2使用時は57.5%が別途報告）を達成。コンテキストウィンドウは最大1Mトークン（ベータ）で、コンパクション機能による実質的に無限の会話が可能。APIの価格は入力$5/1M、出力$25/1Mトークン。

    コスト効率を重視する場合はClaude Sonnet 4.6が選択肢となる。SWE-bench Verifiedで79.6%と、Opus 4.6からわずか1.2%の差で、APIは入力$3/1M、出力$15/1Mと約半額である。Agent Teamsでワーカーエージェントにはこのモデルを使い、リードエージェントにのみOpus 4.6を使うというコスト最適化が実践されている。

    Chapter 05 — Architecture

    アーキテクチャ比較：サンドボックスとセキュリティ

    AIエージェントにコード実行やファイル操作を許可する以上、サンドボックス設計とセキュリティモデルは極めて重要である。両者は根本的に異なるアーキテクチャを採用しており、そのトレードオフを理解することが実運用で不可欠となる。

    OpenAI Codex：二層セキュリティモデル

    Codexのセキュリティはサンドボックスモードと承認ポリシーの二層で構成される。クラウドモードでは、OpenAI管理の隔離コンテナ内でタスクが実行される。セットアップフェーズ（ネットワークアクセスあり、依存関係インストール可能）とエージェントフェーズ（デフォルトでオフライン）の二段階ランタイムモデルを採用し、セットアップ時に設定されたシークレットはエージェントフェーズ開始前に削除される。

    ローカル（CLI/IDE拡張）では、OS レベルの機構でサンドボックスが実施される。macOSではSeatbeltポリシー（sandbox-exec）、LinuxではLandlock + seccomp（またはオプションでbwrap）、WindowsではWSL経由またはネイティブWindowsサンドボックスが使用される。デフォルトのworkspace-writeモードでは、ネットワークアクセスは無効、書き込み権限はアクティブなワークスペースに限定される。.gitディレクトリは読み取り専用として保護される。

    承認ポリシーは柔軟に設定可能で、--full-auto（ワークスペース内は自動、外部は承認要求）から--dangerously-bypass-approvals-and-sandbox（通称--yolo：全アクセス）まで段階がある。また、OpenTelemetry（OTel）によるオプトイン監査ロギングがサポートされており、エンタープライズ環境でのコンプライアンスに対応する。

    Claude Code：ローカル実行＋権限管理

    Claude Codeはローカルマシンで動作し、開発者のファイルシステムとシェル環境に直接アクセスする。Codexのようなクラウドサンドボックスによる分離はなく、代わりに権限システムと承認プロンプトによる制御を採用する。デフォルトでは、ファイル編集やコマンド実行の前に開発者の承認を求める。--dangerously-skip-permissionsフラグで自動承認も可能だが、名前が示す通り注意が必要である。

    Git worktreeによるエージェント分離がAgent Teamsで採用されており、各エージェントが独立したworktreeで作業することでファイルコンフリクトを防ぐ。Hooksシステムにより、ファイル書き込みやコマンド実行の前後にカスタムスクリプトを実行でき、決定論的な品質ゲートを設けることができる。

    Chapter 06 — Context & Memory

    コンテキストウィンドウ・長期記憶・指示ファイル

    コンテキストウィンドウの管理は、AIコーディングエージェントの実用性を決定づける最重要要素の一つである。大規模なコードベースを扱う際、エージェントが「忘れる」ことは致命的な品質低下を招く。両者はこの課題に対して異なるアプローチを採用している。

    コンテキストウィンドウの比較

    GPT-5.3-Codexは400Kトークン（入力272K + 出力128K）のコンテキストウィンドウを提供する。Codexの特徴的なアプローチはdiff-based forgetting（差分ベースの忘却）である。古いコンテキストを要約にコンパクションするのではなく、陳腐化したコンテキストを「差分」として除去し、変更デルタのみを保持する。これにより、要約ベースのアプローチよりもコードベースの構造的理解を維持できるとされる。

    Claude Opus 4.6は最大1Mトークン（ベータ）のコンテキストウィンドウを持ち、Sonnet 4も1Mトークンに拡大されている。さらにコンパクション機能（サーバーサイドの自動コンテキスト要約）により、実質的に無限の会話が可能である。コンテキストがウィンドウの限界に近づくと、APIが自動的に古い部分を要約し、新しいメッセージのための空間を確保する。ただし、初期の指示が要約過程で失われる可能性があるため、重要な指示はCLAUDE.mdに記載することが推奨される。

    AGENTS.md vs CLAUDE.md

    両者ともリポジトリ内に配置するマークダウンファイルでエージェントに指示を与える仕組みを持つ。CodexのAGENTS.mdは、コードベースのナビゲーション方法、テスト実行コマンド、プロジェクトの標準プラクティスを記述する。ネストして階層的に配置でき、サブディレクトリごとに異なる指示を設定可能。Claude CodeのCLAUDE.mdは同様の役割を果たし、プロジェクト固有の指示、ワークフロー、コーディング規約、使用するMCPツール等を定義する。

    両者の重要な違いは、Claude Codeには自動メモリ（Auto-memory）機能がある点である。セッション中に学習したプロジェクトの文脈を自動的に保存し、次のセッションで参照できる。MEMORY.mdファイルとしてローカルに保存され、Agent Teamsではチームメンバー間で共有可能な形式で管理される。

    Chapter 07 — Tool Integration

    ツール統合：MCP・bash・ファイル・Git

    AIコーディングエージェントの実用的な力は、どのようなツールとどの程度深く統合できるかに大きく依存する。ここでは、MCP（Model Context Protocol）、bash実行、ファイル操作、Git操作という4つの主要領域について両者の実装差異を分析する。

    MCP（Model Context Protocol）

    MCPはAnthropicが主導するオープンスタンダードであり、AIモデルを外部ツールに接続するためのプロトコルである。2025年11月に正式発表され、以降急速にエコシステムが拡大した。Claude Codeでは創設者としてのアドバンテージがあり、MCPは設計の中核に組み込まれている。設定ファイルやCLAUDE.md内でMCPサーバーを指定することで、データベース接続、API呼び出し、カスタムツール等をシームレスに統合できる。

    CodexもMCPをサポートしている。config.toml内でMCPサーバーを設定可能で、@メンションによるMCPショートカットや、承認制御の統合など独自の工夫がある。ただし、MCPの追加はコンテキスト消費量を増やすため、不要なMCPサーバーを無効化することが推奨されている。

    bash実行とファイル操作

    Codexはローカルモードでサンドボックス内のbash実行をサポートし、テストハーネス、リンター、型チェッカーなどのコマンドを実行できる。クラウドモードでは各タスクのコンテナ内でコマンドが実行される。ファイル操作はread/write/patchの形式で行われ、パッチベースのワークフロー（git diff/git apply）が推奨される。

    Claude Codeはローカルマシンのシェルに直接アクセスし、任意のコマンドを実行できる。ファイル操作はRead/Write/Editの形式で、インラインdiffによる変更提示が特徴的である。Hooksシステムにより、bash実行の前後にカスタムスクリプトを挟むことができ、リンティングやフォーマッティングの自動適用が可能。

    Git操作

    両者ともGit操作を深くサポートするが、アプローチが異なる。Codexはクラウドモードでリポジトリのスナップショットをコンテナに読み込み、作業完了後にコミットを作成してGitHub PRとして提出する。CLIモードではローカルGitリポジトリを直接操作するが、.gitディレクトリは読み取り専用として保護される。Claude CodeはローカルGitリポジトリとネイティブに統合し、ブランチ作成、コミット、PR作成を直接実行する。Agent Teamsでは各エージェントがGit worktreeで分離されて作業し、マージ時のコンフリクトを最小化する。

    初期セットアップと立ち上がり

    Codex CLIはRust製のゼロ依存バイナリであり、インストールは極めてシンプルである。ChatGPTアカウントでサインインすれば自動的にAPIキーが設定される。AGENTS.mdがなくてもそこそこの結果を出す「箱出し」の良さが特徴で、設定なしでも実用的な出力を得られる。

    Claude Codeもcurl -fsSL https://claude.ai/install.sh | bash一行でインストール可能で、セットアップ自体は簡単である。しかし、Claude Codeの真の力は設定を詰めることで発揮される。CLAUDE.mdでプロジェクト固有の指示を記述し、Hooksでカスタム自動化を組み、MCPサーバーで外部ツールを統合する。この「設定税（Configuration Tax）」は初期投資が必要だが、プロジェクトが大きくなるほどリターンが大きい。

    日常的なコーディング体験

    Codexは「速さ」と「独立性」に秀でる。GPT-5.3-Codex-Sparkは1,000+ tok/secで応答し、タスクを投げてから結果が返るまでの体感が軽い。クラウドモードでは複数タスクを並列実行できるため、朝にタスクを一括投入し、コーヒーを飲みながら結果を待つワークフローが可能。しかし、同じプロンプトに対して結果がランブル（variability）する傾向があり、一貫性では課題がある。

    Claude Codeは「深さ」と「一貫性」に秀でる。思考過程を可視化し（think out loud）、明確な理由付けとともにコードを生成する。同じプロンプトに対する出力の再現性が高く、「プランに忠実に従う」傾向がある。トークン消費量がCodexの3-4倍程度になることがあるが、これはより詳細な推論と説明を行うためである。ただし、使用制限に達するとタスクの途中で停止する「リミットウォール」が発生する場合がある。

    Chapter 09 — Large-Scale Refactoring

    大規模リファクタリング・マルチファイル編集

    大規模なリファクタリングやマルチファイル編集は、AIコーディングエージェントの真価が問われる領域である。単一ファイルの修正とは異なり、コードベース全体の構造理解、一貫性の維持、副作用の予測が求められる。

    Codexのアプローチ

    Codexのクラウドモードは大規模リファクタリングに適した設計を持つ。リポジトリ全体がサンドボックスに読み込まれ、ファイルの読み書きとテスト実行が自由に行える。GPT-5.2-Codexでコンテキストコンパクションが導入されて以来、長時間の大規模変更が改善された。ただし、コンテキストスイッチ（複数ファイル間の行き来）で追跡を失う傾向があり、2000行を超えるファイルでもスムーズに処理できるが、40ファイルを横断するような変更では一貫性が低下する場合がある。

    Claude Codeのアプローチ

    Claude Codeの1Mトークンコンテキストは大規模リファクタリングで大きなアドバンテージとなる。さらに、Agent Teamsの「分割統治」アプローチが威力を発揮する。例えば、認証モジュールのリファクタリングをリードエージェントが計画し、フロントエンド、バックエンド、テストをそれぞれ別のエージェントが担当するワークフローが可能である。各エージェントはGit worktreeで分離されて作業するため、同時編集によるコンフリクトを回避できる。

    実際の実績として、16個のClaudeエージェントがRustで100K行のCコンパイラを構築し、Linuxカーネル6.9をコンパイルした事例が報告されている（GCC tortureテストの99%パス率、API費用約$20K）。これはAgent Teamsが単なるCRUDスキャフォールディングではなく、真に複雑なエンジニアリングタスクに対応できることの証明である。

    Chapter 10 — Debug, Test & CI/CD

    デバッグ・テスト実行・CI/CD連携

    テスト実行とCI/CDパイプラインとの統合は、エンタープライズ開発において特に重要な領域である。

    テスト実行

    Codexはcodex-1の強化学習において「テストが通るまで反復的に実行する」能力が特に強化されている。AGENTS.mdに記述されたテストコマンドを自動的に実行し、テスト失敗時は修正を試みてから再実行するループを組む。GPT-5.3-Codexでは、テスト失敗の根本原因分析も改善された。

    Claude CodeもCLAUDE.mdに記述されたテストコマンドを実行し、テスト結果に基づいてコードを修正する。HooksシステムのTaskCompletedイベントにテスト実行スクリプトを設定することで、タスク完了前に自動的にテストを実行し、失敗した場合はフィードバックを返して再作業させることができる。

    CI/CD統合

    Codexは--sandbox read-only --ask-for-approval neverモードでCI環境での非対話的実行をサポートする。APIキーベースの認証でCI/CDパイプラインに組み込み可能。GitHub統合では自動コードレビュー機能が提供され、@CodexタグでPRレビューをトリガーできる。

    Claude CodeはGitHub Actionsとの公式統合を提供し、@claudeメンションでPRやIssueに対してAIを操作できる。GitLab CI/CDとの統合も公式サポートされている。Slack統合により、バグ報告をSlackスレッドから直接PRに変換するワークフローも実現できる。

    チーム開発・エージェント協調

    2026年2月に両者がマルチエージェントワークフローをサポートしたことは、AIコーディングエージェントの歴史における転換点である。「タスクごとに専用のコンテキストウィンドウを持つ」というプリミティブが、エージェントプログラミングの最初の永続的な基盤として確立されつつある。

    Codex App：プロジェクトベースのマルチエージェント

    2026年2月2日に発表されたCodex macOSアプリは「エージェントのためのコマンドセンター」として設計されている。開発者はプロジェクトごとにタスクを整理し、複数のエージェントを並列に実行できる。各エージェントは独立したクラウドサンドボックスで動作し、互いのコンテキストを汚染しない。Skills（再利用可能な指示パッケージ）とAutomations（自動化ルール）により、定型的なワークフローを効率化できる。

    ただし、Codexのマルチエージェントは本質的に「独立並列」であり、エージェント間の通信やタスク依存関係の管理は限定的である。各エージェントは独立して作業し、結果を開発者が統合する。

    Claude Code Agent Teams：協調型マルチエージェント

    Claude CodeのAgent Teamsはより深い協調モデルを提供する。リードセッションがチームを統括し、タスクの割り当て、進捗監視、結果の統合を行う。チームメイトは独立したコンテキストウィンドウを持ちながら、以下の機能で連携する：

    共有タスクリストには依存関係の追跡機能があり、前提タスクが完了するまでブロックされるタスクを自動管理する。ダイレクトメッセージングにより、チームメイト同士が直接情報を交換できる。ブロードキャスト機能で全チームメイトに一斉通知も可能。タスクのクレームにはファイルロックが使用され、複数エージェントが同一タスクを取り合う競合を防止する。

    さらに、リードエージェントによるプラン承認の仕組みがあり、チームメイトの実装計画をリードがレビュー・承認してから作業を開始させることで、品質を担保する。HooksのTeammateIdleやTaskCompletedイベントでカスタム品質ゲートを設けることも可能である。

    Chapter 12 — Pricing & Cost

    料金体系・コスト効率の徹底比較

    料金体系は両者の利用判断において最も現実的な要素の一つである。2026年3月時点の料金を徹底的に比較する。

    サブスクリプション比較

    OpenAIは3つのサブスクリプション階層を提供している。ChatGPT Go（$8/月）は限定的なCodex利用が可能な新しいエントリーティア。ChatGPT Plus（$20/月）は5時間あたり45-225ローカルメッセージ、10-60クラウドタスク、週10-25コードレビュー。ChatGPT Pro（$200/月）は5時間あたり300-1,500ローカルメッセージ、50-400クラウドタスク、週100-250コードレビュー。Enterprise/Eduプランは別途。使用制限を超えた場合はクレジットを購入して延長可能。

    Anthropicは3つの主要プランを提供している。Claude Pro（$20/月、年額$17/月）は標準的な使用制限。Claude Max 5x（$100/月）はProの5倍の使用量。Claude Max 20x（$200/月）はProの20倍の使用量。Team（$30/ユーザー/月）およびEnterpriseプランも利用可能。

    エコシステム・コミュニティ

    AIコーディングエージェントの価値は、ツール単体の能力だけでなく、周囲のエコシステムとコミュニティの厚さにも依存する。

    オープンソースとコントリビューション

    Codex CLIはApache-2.0ライセンスで完全にオープンソースであり、365名のコントリビューターが参加し、10ヶ月間で553リリース（平均1.8/日）を達成している。Rust移行後はコミュニティの技術的貢献がさらに活発化した。一方、Claude Codeはプロプライエタリ製品であるが、51名のコントリビューターによるIssueトラッカーとフィードバックは活発で、1日に複数回のリリースを行う。基盤モデルは両者ともプロプライエタリである。

    Agent Skills エコシステム

    2025年末から「Agent Skills」というオープンスタンダードが急速に普及した。SKILL.mdファイルとしてパッケージ化された再利用可能な指示セットで、Claude Code、OpenAI Codex、Gemini CLI、その他の互換ツール間で共有できる。agentskills.ioを中心としたコミュニティが形成され、Web開発、テスト、デプロイメントなど様々なスキルが公開されている。

    MCP エコシステム

    MCPはAnthropicが主導するオープンスタンダードとして、急速にエコシステムが拡大している。データベース接続、API統合、監視ツール、ブラウザ操作など、多数のMCPサーバーがコミュニティによって開発されている。WarpGrep v2のようなMCPサーバーはClaude Code、Codex、Cursor等で横断的に使用可能で、SWE-bench Proスコアを49.8%から57.5%に向上させた事例もある。

    コミュニティの規模

    GitHubスターではClaude Codeが71,500でリード、Codexが62,365で追う。VS Codeマーケットプレイスではそれぞれ5.2M、4.9Mインストール。Reddit上ではr/ClaudeAIおよびr/ClaudeCodeが活発で、Codex関連はr/codexとr/ChatGPTCodingが主要コミュニティ。開発者の日常会話ではClaude Code支持者が「一貫性と品質」を、Codex支持者が「速度と並列性」をそれぞれ高く評価する傾向がある。

    Chapter 15 — Future Outlook

    将来展望 — 両者はどこに向かうのか

    2026年3月時点で、AIコーディングエージェント市場は年間数十億ドル規模に成長しており、OpenAIとAnthropicの両社はこの領域を主要な成長ベクトルと位置づけている。Anthropicの評価額は$380Bに達し、ARRは$14Bを記録。OpenAIはCodexを非Nvidiaハードウェア（Cerebras WSE-3）に展開し始めている。両者の戦略的方向性を展望する。

    OpenAI Codexの方向性

    OpenAIはCodexを「コーディングエージェント」から「汎用コンピュータ操作エージェント」へと進化させようとしている。GPT-5.3-CodexがOSWorldで64.7%を達成したことは、プレゼンテーション作成、スプレッドシート分析、デスクトップ操作など、コーディング以外の知的作業への展開を示唆する。「リアルタイムペアリング」と「タスク委任」の2つのモードの収束が計画されており、IDEからCI、イシュートラッカーまで、開発者のあらゆるツールとの深い統合が目指されている。

    サイバーセキュリティも重要な戦略軸である。GPT-5.3-Codexは初めて「サイバーセキュリティ高能力」に分類され、脆弱性発見能力が大幅に向上した。$10Mのサイバー防御API クレジットプログラムやAardvarkセキュリティ研究エージェントの開発は、セキュリティ分野での優位性確立を目指す動きである。

    Claude Codeの方向性

    Anthropicは「開発者のワークフローのあらゆる接点にClaude Codeを埋め込む」戦略を加速している。ターミナル、VS Code、JetBrains、デスクトップ、Web、Slack、GitHub Actions、GitLab CI/CD、Chrome、iOSアプリ、Remote Controlと、すでにほぼ全方位をカバーしている。Agent Teams（研究プレビュー）の正式リリースとさらなる機能拡充が予想される。

    Agent SDKの提供は特に重要な動きである。Claude Codeと同じツール・エージェントループをPythonやTypeScriptから利用可能にすることで、開発者が独自のAIコーディングエージェントを構築できるプラットフォームとしての位置づけを強化している。SemiAnalysisの分析では、Claude Codeのコミット数は2026年末までに全公開GitHubコミットの20%以上に達すると予測されている。

    収束と分化

    METR（AI Agent Task Horizon）のデータによると、AIエージェントのタスクホライズンは4-7ヶ月ごとに倍増しており、2019年の1分間から2026年には複数時間のタスクに対応するまでに成長した。この指数関数的な成長が続く限り、両者の能力は急速に向上し続ける。

    今後の競争軸は、単純なコード生成能力から、マルチエージェントオーケストレーション、自律的なプロジェクト管理、セキュリティ、コスト効率、そしてエンタープライズガバナンスへとシフトすると予測される。両者は現在異なるアーキテクチャを採用しているが、機能面での収束は不可避である。最終的に勝者を決めるのは、開発者エクスペリエンスの質と、エコシステムの深さであろう。
