Section 01

Gemini Code Assistとは

Google が提供する AI パワードのコーディングアシスタント。Gemini の大規模言語モデルを活用し、ソフトウェア開発ライフサイクル全体を支援します。

🤖概要

Gemini Code Assist は、Google が提供する AI パワードのコーディングアシスタントです。もともと「Duet AI for Developers」として 2023 年に登場し、2024 年に Gemini Code Assist へと名称変更されました。Gemini 2.5 モデル（将来的には Gemini 3 も対応予定）を基盤として、IDE 上でのコード補完、チャットベースの対話的なコーディング支援、コード生成、テスト生成、デバッグ支援、コードレビューなど、開発者の生産性を飛躍的に向上させる多彩な機能を備えています。

個人開発者向けの無料版（Gemini Code Assist for individuals）から、チームや企業向けの Standard / Enterprise エディションまで、さまざまな規模の開発ニーズに対応しています。特筆すべきは、無料版でも月間 180,000 回のコード補完と、1 日あたり 240 回のチャットリクエストという非常に高い利用上限が設定されている点です。これは競合ツールと比較しても圧倒的に寛大な無料枠です。

📊Google Cloud 統合のメリット

Gemini Code Assist は単なる IDE プラグインではなく、Google Cloud エコシステム全体に深く統合されています。Standard エディション以上では Firebase でのモバイル・Web アプリ開発支援、BigQuery でのデータ分析支援（SQL 生成・最適化）、Cloud Run でのコード補完、Colab Enterprise でのノートブック支援、Database Studio での SQL クエリ生成など、クラウドサービス横断で AI アシスタンスを受けられます。

Enterprise エディションではさらに Apigee での API 管理支援、Application Integration での自動化フロー生成、Gemini Cloud Assist による Google Cloud コンソール全体での AI 支援（インフラ管理、診断、コスト最適化）が加わります。これにより、コーディングだけでなく、アプリケーションの設計・デプロイ・運用まで、SDLC（ソフトウェア開発ライフサイクル）全体をカバーする包括的な AI 支援が実現します。

💡 ポイント

Gemini Code Assist は 2025 年 10 月に従来の「ツール」方式が廃止され、より高度な「エージェントモード（Agent Mode）」に移行しました。エージェントモードでは、マルチステップの複雑なタスクを自律的に実行でき、MCP（Model Context Protocol）サーバーとの連携も可能です。

Section 02

Julesとは

Google Labs が開発した非同期 AI コーディングエージェント。GitHub と統合し、バックグラウンドで自律的にコーディングタスクを実行します。

⚡Julesの概要

Jules は、Google Labs が開発した非同期（asynchronous）の AI コーディングエージェントです。2024 年 12 月に限定プレビューとして初登場し、2025 年 5 月に Google I/O でパブリックベータ版が公開され、2025 年 8 月に正式にベータを卒業して一般提供（GA）が開始されました。Gemini Code Assist が IDE 内での対話型アシスタントであるのに対し、Jules はよりエージェンティックな存在として設計されています。

Jules の最大の特徴は「非同期性」です。開発者がタスクを指示すると、Jules はそれを Google Cloud 上のセキュアな仮想マシン（VM）にクローンされたリポジトリ上で自律的に実行します。開発者はその間、他の作業に集中でき、タスクが完了するとブラウザ通知で知らされます。Jules はコードの変更計画（プラン）を提示し、開発者の承認を得てから変更を適用します。変更結果は GitHub のプルリクエストとして提出されます。

GitHub ネイティブ統合

GitHub リポジトリに直接接続し、既存のワークフローの中で動作します。新しいツールを学ぶ必要はなく、いつもの Git フローで Jules の成果物を取り込めます。

🔄

並列タスク実行

複数のタスクを同時に実行可能です。各タスクは独立した Cloud VM 上で動作するため、互いに干渉しません。無料プランで 3 並列、Pro で 15 並列、Ultra で 60 並列まで対応します。

👁️

透明なワークフロー

コード変更前に必ず計画（プラン）と推論過程を提示します。開発者は実行前・実行中・実行後にプランを修正でき、コードベースの主導権を維持できます。

🎧

音声チェンジログ

最近のコミット内容を音声形式のチェンジログとして生成する機能を搭載。移動中やマルチタスク時にプロジェクトの変更履歴をオーディオで確認できます。

📄AGENTS.md ファイル

Jules はリポジトリのルートに配置された AGENTS.md ファイルを自動的に読み取ります。このファイルにプロジェクトの構造、コーディング規約、依存関係、ビルド手順などを記述しておくことで、Jules がより正確で文脈に適したプランやコードを生成するようになります。これは、新しいチームメンバーへのオンボーディングドキュメントのような役割を果たし、AI エージェントの出力品質を大幅に向上させます。

💡 Gemini Code Assist との違い

Gemini Code Assist は IDE 内でリアルタイムに動作する対話型のアシスタント（Agent Mode 含む）であり、Jules はバックグラウンドで非同期に動作する自律型エージェントです。Gemini CLI は「ターミナルで今フォーカスしているタスクの協力者」、Jules は「背後で独立して動く自律的な相棒」と位置づけられています。両者は補完的な関係にあり、2025 年 10 月には Gemini CLI から Jules タスクを起動・管理できる Jules 拡張機能もリリースされました。

Section 03

主な機能

Gemini Code Assist と Jules が提供する主要機能を詳しく解説します。

⌨️コード補完と生成

Gemini Code Assist のコード補完は、開発者がコードを書いている最中にリアルタイムで次に書くべきコードを提案する機能です。単なる行単位の補完ではなく、Gemini の大規模コンテキストウィンドウを活用して、現在のファイルだけでなくプロジェクト全体の文脈を理解した上で提案を行います。関数全体の生成、コードブロックの生成、コメントからのコード自動生成にも対応しています。

さらに「Next Edit Predictions（次の編集予測）」機能（プレビュー）では、開発者が次にどの箇所を編集しようとしているかを予測し、プロアクティブに提案を行います。これにより、繰り返しの編集作業（関数名のリネームに伴う複数箇所の修正など）を大幅に効率化できます。

無料版では月間 180,000 回のコード補完が利用可能で、これは 1 日あたり約 6,000 回に相当します。Standard / Enterprise ではさらに高い上限が設定されています。

💬チャット & 対話型アシスタント

IDE 内のチャットパネルで、自然言語による対話形式でコーディング支援を受けられます。開いているファイルのコンテキストを自動的に認識し、コードの説明、デバッグ支援、リファクタリング提案、ユニットテスト生成などを行います。「Write unit tests for my code」「Help me debug my code」「Make my code more readable」といったプロンプトを入力するだけで、即座に実用的なレスポンスが返されます。

チャットではソースサイテーション（出典引用）機能も提供されます。Gemini が既存のオープンソースコードから長い引用を行った場合、その出典情報を明示し、ライセンス要件への準拠を支援します。

🤖エージェントモード（Agent Mode）

2025 年後半に導入されたエージェントモードは、Gemini Code Assist の最も強力な機能の一つです。従来の単純なコマンド＆レスポンス型のやり取りを超え、マルチステップの複雑なタスクを自律的に計画・実行する推論エージェントとして機能します。

開発者がゴールを記述すると、Gemini がまず詳細な計画を提案します。開発者がその計画を確認・承認した後、Gemini はシステムツール（ファイルの読み書き、ターミナル実行、grep など）や MCP サーバーを活用しながら、自律的にコード変更を行います。VS Code のエージェントモードは内部的に Gemini CLI のパワーを活用しており、IDE の使い慣れた UI から高度なエージェンティック操作が可能です。

MCP（Model Context Protocol）のサポートにより、外部ツールやサービスとの連携が拡張可能です。データベースクライアント、CI/CD パイプライン、ドキュメント検索システムなどを MCP サーバーとして接続し、エージェントモードから利用できます。

🔍コードレビュー（GitHub 統合）

Gemini Code Assist for GitHub は、GitHub Marketplace からインストール可能なアプリで、プルリクエストの自動レビューを提供します。PRが作成されると自動的にレビューが実行され、以下の機能が提供されます。

PR サマリーの自動生成：変更内容の概要を自動的に要約し、レビュワーが変更の全体像を素早く把握できるようにします。インラインコードレビュー：個別のコード行に対して問題点の指摘やコミット可能な修正提案を行います。対話的なフィードバック：PRのコメント欄で /gemini タグを使って質問やフォローアップが可能です。カスタマイズ可能なレビューガイドラインを設定することで、チームのコーディング規約に沿ったレビューを行わせることもできます。

📟Gemini CLI

Gemini CLI は、2025 年 6 月にリリースされたオープンソース（Apache 2.0 ライセンス）のターミナルベース AI エージェントです。Gemini のパワーを直接ターミナルに持ち込み、コードの読み書き、デバッグ、タスクの自動化を行えます。VS Code 内のエージェントモードは Gemini CLI のサブセットとして動作しており、両者はシームレスに連携します。

主な機能として、大規模コードベースのクエリと編集、PDF・画像・スケッチからのアプリ生成（マルチモーダル機能）、Google 検索によるグラウンディング、MCP サーバーのサポート、メモリモジュール（/memory コマンド）、YOLO モード（確認なしの連続実行）などがあります。Cloud Shell にはプリインストールされているため、追加セットアップなしですぐに利用を開始できます。

✏️スマートアクションとコマンド

コードを選択して右クリックから呼び出せるスマートアクション（VS Code / JetBrains / Android Studio 対応）と、スラッシュコマンド / でトリガーするスマートコマンド（VS Code）が用意されています。これにより、テスト生成（/tests）、コード説明、ドキュメント生成、コード修正、リファクタリングなどの頻繁に使う操作を素早く実行できます。

Section 04

基盤モデル

Gemini Code Assist と Jules を支える AI モデルについて解説します。

🧠Gemini 2.5 Pro

Gemini Code Assist と Jules の中核を担うのがGemini 2.5 Proモデルです。Google DeepMind が開発したこのモデルは、100 万トークン以上のコンテキストウィンドウを持ち、大規模なコードベース全体を一度に理解・処理する能力を備えています。特にコーディングタスクにおいて、高度な推論能力（thinking capabilities）を発揮し、複雑なマルチファイル変更やアーキテクチャ設計の提案にも対応します。

Jules では Gemini 2.5 Pro の高度な思考能力を活用して、コーディング計画（プラン）を生成しています。これにより、単純なコード補完を超えた、プロジェクト全体の文脈を考慮した高品質なコード出力が可能になっています。

⚡Gemini Flash & 今後の展開

コード補完などの低レイテンシが求められるタスクでは、Gemini Flashシリーズのモデルが使用されています。Flash モデルは Pro モデルに比べてレスポンスが高速で、リアルタイムのコード補完体験を実現するために最適化されています。

今後については、Gemini 3 モデルの導入が予定されています。Jules の有料プラン（Pro / Ultra）では Gemini 3 Pro への優先アクセスが予告されており、さらに高品質なコーディング支援が期待されます。公式ページでも「Gemini 3 coming soon」と明記されており、Gemini Code Assist でも順次 Gemini 3 ベースに移行していく見込みです。

Section 05

対応IDE・インターフェース

Gemini Code Assist はマルチ IDE 対応。主要な開発環境をすべてカバーします。

🟦

Visual Studio Code

最もフル機能が利用できる IDE。コード補完、チャット、Agent Mode、Gemini CLI 統合、MCP サーバー、スマートアクション・コマンドのすべてに対応。VS Code Marketplace からワンクリックでインストール可能です。

🟧

JetBrains IDEs

IntelliJ IDEA（Community / Educational / Ultimate）、PyCharm（Community / Professional）、WebStorm、GoLand、PhpStorm、Rider、RubyMine、CLion、DataGrip に対応。JetBrains Marketplace からプラグインをインストールして使用します。

🤖

Android Studio

Android 開発専用の Gemini 統合。コード補完・生成に加えて、変数・メソッド名の提案、Compose プレビュー作成、画像からの UI 構築、コミットメッセージ生成など、Android 特化の機能を多数搭載しています。

☁️

Cloud Shell Editor / Cloud Workstations

Google Cloud のブラウザベース開発環境。Cloud Shell には Gemini CLI がプリインストールされており、追加セットアップなしですぐに利用開始可能。Cloud Workstations でも Gemini Code Assist を利用できます。

🖥️Gemini CLI（ターミナル）

IDE を離れてターミナルで作業したい開発者向けに、Gemini CLI が用意されています。npm でインストールするか、Cloud Shell で直接利用できます。VS Code のエージェントモードと同じ Gemini CLI エンジンを使用しており、MCP サーバー、メモリ、Web 検索、ファイル操作、シェル実行などのツールを活用した高度なタスク実行が可能です。オープンソース（Apache 2.0）なので、カスタマイズや拡張も自由に行えます。

🌐対応プログラミング言語

Gemini Code Assist は「パブリックドメインにあるすべてのプログラミング言語」をサポートすると公式に謳っています。特に高品質なサポートが確認されている言語は、Bash、C、C++、C#、Dart、Go、Java、JavaScript、TypeScript、Kotlin、PHP、Python、Ruby、Rust、Swift などの主要言語です。さらにチャットインターフェースでは、日本語を含む多数の自然言語での対話にも対応しています。

Section 07

Google Cloud・Firebase・Android Studio 連携

Google のエコシステム全体にわたる AI 支援の統合を詳しく解説します。

🔥Firebase 連携

Firebase では「Gemini in Firebase」として、Firebase コンソール内で AI アシスタンスが利用できます。Firebase プロダクトに関する深い知識、ベストプラクティス、トラブルシューティングの専門知識を活用した対話型支援のほか、コードの生成・リファクタリング・デバッグを自然言語で行えます。

Crashlytics との連携ではアプリクラッシュの要約・分析・トラブルシューティング手順の提案が行われ、Cloud Messaging / In-App Messaging ではキャンペーンの要約・分析・パフォーマンス改善の提案が可能です。Data Connect ではスキーマの自然言語生成や GraphQL クエリ・ミューテーションの生成に対応しています。

さらに Firebase Studio（旧 Project IDX）では、Gemini Code Assist のエージェント機能が統合されたフルスタックのクラウド開発環境が提供されています。ブラウザから直接、Gemini の AI 支援を受けながらアプリケーション開発を行えます。

📱Android Studio 連携

Android Studio では「Gemini in Android Studio」として、Android 開発に特化した AI 支援が提供されています。コード補完・生成・変換に加えて、変数・メソッド・クラスの命名提案、コードドキュメント生成、コミットメッセージ作成、Jetpack Compose プレビューの作成、画像からの UI 構築などの Android 開発特有の機能を備えています。

企業向けには「Gemini in Android Studio for businesses」として、法人のプライバシー・セキュリティ・管理ニーズに対応した構成も提供されています。これは Gemini Code Assist Standard / Enterprise のライセンスで利用可能です。

📊BigQuery 連携

BigQuery Studio では、自然言語から SQL を生成する機能、SQL の説明・補完・最適化機能、スキーマを考慮したコンテキスチュアルなコード生成が利用できます。Python コードの生成・補完にも対応しており、Data Canvas や Data Preparation などの基本的な AI 機能は追加費用なしで全 BigQuery エディションで利用可能です。

Standard / Enterprise ライセンスでは、テーブルのメタデータから自動的にインサイト（分析クエリのライブラリ）を生成する Data Insights や、自動メタデータ生成などの高度な機能が追加されます。

🔗その他の Google Cloud 連携

Cloud Run：Cloud Run のソースエディタ内でコード補完が利用でき、現在のコンテキストに関連するコード提案を受けられます。Colab Enterprise：ノートブック内で Python コードの生成・補完が可能で、データサイエンスやML ワークフローを加速します。Database Studio：自然言語から SQL を生成し、スキーマ対応のコード補完、クエリの最適化・説明を行います。

Apigee（Enterprise のみ）：API 仕様の設計・作成をエンタープライズコンテキストを使って加速します。API Hub でのスマート検索やポリシーの説明機能も備えています。Application Integration（Enterprise のみ）：自動化フローの AI 支援ビジュアルエディタ、コンテキストを埋め込んだ自動化作成支援、ドキュメント生成・改善を提供します。Gemini Cloud Assist（Enterprise のみ）：Google Cloud コンソール全体で Infrastructure as Code の生成、診断・調査、コスト最適化、IAM 支援などの高度な AI 支援を提供します。

Section 08

セキュリティ・プライバシー

企業が安心して導入できるセキュリティ体制とデータガバナンスを解説します。

🔒データプライバシー

Gemini Code Assist の Standard / Enterprise エディションは、Google のAI/ML プライバシーコミットメントに準拠しています。最も重要なポイントとして、ユーザーのプロンプト（入力）やレスポンス（出力）がモデルのトレーニングデータとして使用されることはありません。Google Cloud の利用規約と Cloud Data Processing Addendum（CDPA）に基づいたデータ処理が行われます。

Jules についても、プライベートコードでモデルのトレーニングは行われず、データは実行環境内で隔離されます。プライバシーバイデフォルトの設計思想が徹底されています。

🔐

暗号化

プロンプトは転送時に暗号化され（encryption in-transit）、保存時にもデフォルト暗号化（encryption at-rest）が適用されます。Google Cloud の堅牢な暗号化インフラを活用します。

🛡️

IP 補償（Indemnification）

Standard / Enterprise エディションでは、Google が生成 AI の知的財産に関する法的リスクを引き受ける IP 補償を提供。トレーニングデータと生成コードの両面で保護されます。

🌐

VPC-SC 対応

Standard / Enterprise エディションでは VPC Service Controls と Private Google Access に対応。ネットワーク境界を設定して、データの流出リスクを最小化できます。

📋

ソースサイテーション

既存のオープンソースコードから長い引用がある場合、出典情報を自動的に提示。ライセンスコンプライアンスへの対応を支援します。

🏢Enterprise セキュリティ機能

Enterprise エディションのコードカスタマイズ機能では、組織のプライベートコードリポジトリ（GitHub、GitLab、Bitbucket）にアクセスしてカスタマイズされた提案を生成しますが、このアクセスと保存はサービス提供に必要な範囲に限定されます。管理者は IAM（Identity and Access Management）を通じてアクセス制御を行い、組織レベルでの有効化・無効化の管理が可能です。

Gemini Code Assist は複数のコンプライアンス認証にも対応しており、企業の規制要件（HIPAA、FedRAMP 等）への準拠を支援する体制が整っています。ただし、生成されたコードのセキュリティ、テスト、有効性については、最終的にはユーザー（開発者）が責任を持つ必要があります。

✅ Jules のセキュリティ

Jules は Google Cloud VM 上のセキュアな環境でコードをクローン・実行します。実行環境はタスクごとに隔離され、プライベートコードがモデルのトレーニングに使用されることはありません。すべての変更は GitHub 上のプルリクエストとして提出されるため、既存のコードレビュー・CI/CD パイプラインを通じた品質管理が維持されます。

Section 09

ユースケース集（10選）

Gemini Code Assist と Jules の実践的な活用シナリオを紹介します。

ユニットテストの自動生成

既存のコードに対してユニットテストを自動生成する最も一般的なユースケースです。Jules にリポジトリを接続し、「Add tests for the parseQueryString function in utils.js」のようにタスクを指示すると、関数の仕様を理解した上でテストコードを生成し、PR として提出します。Gemini Code Assist のチャットやスマートアクションからも同様の操作が可能です。

バグの自動修正

GitHub Issues に報告されたバグを Jules に割り当て、原因の特定から修正コードの作成までを自動化できます。Jules はコードベース全体を理解した上で変更計画を立て、修正パッチを PR として提出します。開発者はプランの確認と PR のレビューに集中できます。

依存関係のアップデート

Node.js、Python、Java などのプロジェクトで、依存ライブラリのバージョンアップに伴うコード変更を Jules に任せることができます。バージョンバンプだけでなく、破壊的変更への対応コードの修正も含めて自動的に処理します。

コードのドキュメント追加

ドキュメントが不足しているコードベースに対して、関数・クラス・モジュールの JSDoc / docstring / README を自動生成します。Jules に「Add comprehensive JSDoc comments to all exported functions in the src/utils directory」と指示するだけで、プロジェクト全体のドキュメント品質を一括で向上させられます。

新機能の実装

既存のコードベースのアーキテクチャとパターンを理解した上で、新しい機能を実装します。たとえば「Add a dark mode toggle to the settings page that persists user preference in localStorage」のような具体的な指示を与えることで、既存のデザインパターンに沿った実装コードを生成します。

コードリファクタリング

レガシーコードの近代化やパフォーマンス改善を行います。Gemini Code Assist のエージェントモードで「Refactor this module to use async/await instead of callbacks」と指示すると、ファイル全体を横断した整合性のあるリファクタリングを実行します。

PR の自動レビュー

GitHub に Gemini Code Assist アプリをインストールすることで、すべての PR に対して自動的にサマリー生成とコードレビューを実行します。バグの潜在リスク、パフォーマンス問題、コーディングスタイルの不整合などを検出し、修正提案を行います。

データベース操作の効率化

BigQuery や Database Studio で自然言語から SQL クエリを生成し、スキーマを考慮した最適なクエリを提案します。「Show me the top 10 customers by revenue in the last quarter, grouped by region」のような自然言語からクエリを即座に生成します。

API の設計・開発

Enterprise エディションの Apigee 統合を活用し、API 仕様（OpenAPI）の設計・作成を AI で加速します。自然言語で API の要件を記述すると、仕様ファイルの自動生成、ポリシーの適用提案、テストケースの生成までを支援します。

モバイルアプリの UI 構築

Android Studio の Gemini 統合を活用し、画像やスケッチから Jetpack Compose の UI コードを生成します。デザインカンプの画像をアップロードするだけで、対応する Compose コンポーネントのコードが生成されるため、デザインから実装への変換を劇的に高速化できます。

Section 10

制限事項・注意点

導入前に知っておくべき制限事項と注意点を整理します。

⚠️Gemini Code Assist の制限事項

出力の正確性：Gemini Code Assist は初期段階の技術であり、もっともらしいが事実上正しくないコードや情報を生成する可能性があります。Google 公式も「すべての出力を使用前に検証することを推奨」と明記しています。生成されたコードのセキュリティ・テスト・有効性はユーザーの責任です。

利用上限：無料版では月間 180,000 回のコード補完（日あたり約 6,000 回）、日あたり 240 回のチャット、Agent Mode / CLI は日あたり 1,000 リクエストという上限があります。活発な開発者では、特に Agent Mode の上限に達する可能性があります。

エージェントモードの制約：2025 年 10 月以降、従来の「ツール」方式は廃止されエージェントモードに移行しましたが、エージェントモードはまだプレビュー段階です。複雑なタスクでは意図しない変更が行われるリスクがあり、慎重なレビューが必要です。

コードカスタマイズの制約：プライベートリポジトリに基づくコード提案のカスタマイズは Enterprise エディションのみの機能です。Standard や無料版では組織固有のコーディングパターンは反映されません。

言語・フレームワークの偏り：主要言語（Python、JavaScript、TypeScript、Go、Java 等）では高品質な提案が得られますが、マイナー言語やニッチなフレームワークでは提案の品質が低下する場合があります。

⚠️Jules の制限事項

GitHub 限定：Jules は現時点で GitHub リポジトリのみに対応しています。GitLab、Bitbucket、Azure DevOps などの他のリポジトリホスティングサービスには対応していません。

地域制限：Jules は Gemini モデルが利用可能な地域でのみ使用できます。一部の国・地域ではアクセスが制限されています。

有料プランのアカウント制限：有料プラン（Pro / Ultra）は現時点で個人 Google アカウント（@gmail.com）のみ対応です。企業の Workspace アカウントからのアップグレードパスは現在準備中です。

タスクの複雑性：非常に大規模なリファクタリングやアーキテクチャレベルの変更では、Jules が適切な計画を立てられない場合があります。また、環境変数やシークレットへのアクセスが制限されるため、外部サービスとの統合テストが必要なタスクには向きません。

年齢制限：Jules を利用するには 18 歳以上である必要があります。Google One ファミリープランの 18 歳未満のメンバーは、プランが有効であっても Jules にアクセスできません。

タスク上限：無料プランでは 1 日 15 タスク（同時 3 タスク）と限定的です。日常的な開発には Pro プラン以上への加入が実質的に必要になります。

⚠️ 重要な注意事項

AI が生成したコードは必ず人間がレビューしてください。特にセキュリティに関わるコード（認証・認可・暗号化・入力バリデーション等）については、AI の提案をそのまま採用せず、セキュリティの専門知識に基づいた検証を行うことが極めて重要です。また、ライセンスコンプライアンスについても、ソースサイテーション機能を活用しつつ、最終的な確認はユーザーが行う必要があります。

始め方ガイド・参考リンク

Gemini Code Assist と Jules を今すぐ始めるためのステップバイステップガイドと参考リソースです。

🚀Gemini Code Assist（無料版）の始め方

ステップ 1：VS Code または JetBrains IDE を用意します。VS Code の場合は VS Code Marketplace で「Gemini Code Assist」を検索し、拡張機能をインストールします。JetBrains IDE の場合は JetBrains Marketplace からプラグインをインストールします。

ステップ 2：IDE の Gemini アイコンをクリックし、Google アカウントでサインインします。個人の Gmail アカウントであれば、追加費用なく「Gemini Code Assist for individuals」が利用可能です。

ステップ 3：プロジェクトを開き、コードを書き始めます。入力中に自動的にコード補完候補が表示されます。Tab キーで候補を受け入れ、Esc キーで候補を無視します。チャットパネルを開いて自然言語で質問することもできます。

ステップ 4（オプション）：エージェントモードを使うには、チャットパネルで複雑なタスクを記述します。Gemini が計画を提案し、承認後に自律的にコード変更を実行します。

⚡Jules の始め方

ステップ 1：jules.google.com にアクセスし、Google アカウントでサインインします。初回はプライバシーに関する通知に同意する必要があります。

ステップ 2：「Connect to GitHub account」をクリックし、GitHub アカウントを連携します。ログインフローを完了し、Jules にアクセスさせるリポジトリ（全リポジトリまたは特定のリポジトリ）を選択します。

ステップ 3：リポジトリセレクターからリポジトリを選び、作業対象のブランチを選択し、プロンプト入力欄にタスクを記述します。たとえば「Add a test for the parseQueryString function in utils.js」のように具体的に書きます。

ステップ 4：「Give me a plan」をクリックすると、Jules が実行計画を生成します。計画を確認・承認すると、Jules がバックグラウンドでタスクを実行します。実行中は自由に他の作業を進められます。

ステップ 5：タスク完了時にブラウザ通知が届きます。結果を確認し、問題なければ Jules が作成した PR をマージします。

プロのヒント：リポジトリのルートに AGENTS.md ファイルを配置し、プロジェクトの構造や規約を記述しておくと、Jules の出力品質が大幅に向上します。

📟Gemini CLI の始め方

Gemini CLI は npm でインストールできます。ターミナルで npm install -g @anthropic-ai/gemini-cli ではなく、正しくは npx https://github.com/google-gemini/gemini-cli またはドキュメントに従って npm install -g @google/gemini-cli でインストールします。Google Cloud Shell を使っている場合はプリインストール済みです。初回起動時に Google アカウントでの認証が求められます。認証後、ターミナル上で自然言語でコーディングタスクを指示できます。

✅ まとめ
