Section 01 — Overview

01Codex（クラウド版）とは何か

OpenAI Codex（コーデックス）は、2025年5月にリリースされたクラウドベースのソフトウェアエンジニアリングエージェントです。ChatGPTのインターフェースからアクセスでき、GitHub上のリポジトリと連携して、コードの生成・バグ修正・テスト実行・プルリクエスト（PR）作成など、ソフトウェア開発に関わる幅広いタスクを自律的に実行します。

従来のAIコーディングアシスタントが「ユーザーの指示にリアルタイムで応答する対話型」だったのに対し、Codexは非同期型の自律エージェントとして設計されています。つまり、タスクを指示したらバックグラウンドで処理が進み、完了後に結果をレビューするワークフローです。これは「同僚にタスクを委任する」感覚に近いものです。

2025年5月の初期リリース以降、Codexは急速に進化しています。2026年2月にはデスクトップアプリ「Codex App」が登場し、複数のエージェントを並列に管理するマルチエージェントワークフローが可能になりました。また、最新のGPT-5.3-Codexモデルにより、コーディングだけでなくコンピュータ操作全般を含む幅広いタスクに対応するエージェントへと進化しています。

Section 02 — Capabilities

02できること

Codexは「AIに開発タスクを丸ごと委任する」ためのエージェントです。単なるコード補完ではなく、リポジトリ全体を理解した上で、ファイルの読み書き・コマンド実行・テスト・PR作成まで一貫して行います。

🔨 コード生成・機能実装

自然言語で「ユーザー認証のAPIエンドポイントを作って」と指示すると、適切なファイルを作成・編集し、必要なインポートや設定ファイルの変更まで含めて実装します。マルチファイルにまたがる変更も自律的に処理します。

🐛 バグ修正・デバッグ

「このテストが失敗する原因を特定して修正して」と依頼すると、テストを実行し、エラーログを分析し、根本原因を特定して修正パッチを作成します。修正後にテストを再実行して成功を確認するところまで自動で行います。

🔀 プルリクエスト作成

タスク完了後、変更をコミットし、GitHub上にPRを自動作成できます。PRには変更内容の説明や関連するテスト結果が含まれ、人間のレビュアーがすぐに確認できる状態になります。

🧪 テスト作成・実行

既存コードに対するユニットテスト・統合テストの作成、テストカバレッジの向上、テストの実行と結果の分析を行います。テストが失敗した場合は自動で修正を試みます。

📖 コードベース質問応答

「Ask」モードで「このプロジェクトの認証フローはどうなっている？」と質問すると、リポジトリ全体を分析して回答します。新規参入メンバーのオンボーディングに非常に有効です。

♻️ リファクタリング

コードの構造改善、命名の統一、重複コードの除去、関数の分割・統合など、大規模なリファクタリングタスクを自律的に実行します。プロジェクトのコーディング規約に沿った形で行われます。

📝 ドキュメント生成

関数やクラスのドキュメントコメント作成、README.mdの更新、APIドキュメントの生成など、コードに付随するドキュメント作成を自動化します。

🔍 コードレビュー

GitHub上のPRに @codex をタグ付けすることで、自動コードレビューを実行できます。バグの可能性、パフォーマンス問題、セキュリティリスクなどを指摘します。

🎨 フロントエンド開発

GPT-5.3-Codexでは、ウェブサイト・ゲーム・アプリのUIを自然言語指示から構築できます。画像生成スキルと組み合わせることで、デザインアセットの作成も可能です。

🔧 Skills による拡張タスク

Codex Appの「Skills」機能により、Figmaデザインの実装、Linear/Slackとの連携、Cloudflare/Vercelへのデプロイ、スプレッドシート・PDF作成など、コード生成を超えた業務を実行できます。

Section 03 — Architecture

03アーキテクチャ

Codexの設計思想を理解することは、効果的に活用するための鍵です。Codexは「安全で隔離された環境内で、コードを自律的に読み・書き・実行する」ことを核としたアーキテクチャを採用しています。

🏗️ サンドボックス環境

各タスクはOpenAIが管理するクラウド上の隔離されたコンテナ内で実行されます。このコンテナにはユーザーのGitHubリポジトリが事前にロードされ、独立した開発環境として機能します。重要な設計上の特徴として、エージェントフェーズではデフォルトでインターネットアクセスが無効化されます。これにより、外部からの悪意ある命令注入（プロンプトインジェクション）のリスクを低減しています。

サンドボックスは2段階のランタイムモデルで動作します。まず「セットアップフェーズ」で、ネットワーク接続を許可して依存関係のインストールなどを実行します。その後の「エージェントフェーズ」ではネットワークを遮断し、リポジトリ内のコードとプリインストールされた依存関係のみを使って作業を行います。

📂 マルチファイル操作

Codexはリポジトリ全体にアクセスできるため、複数のファイルにまたがる変更を一貫して行えます。例えば、APIのエンドポイントを追加する場合、ルーティングファイル、コントローラー、モデル、テストファイル、さらにドキュメントまで、関連するすべてのファイルを同時に変更します。各タスクは独立したサンドボックスで実行されるため、複数の変更が競合することはありません。

🔄 自律実行ループ

Codexの中核は「エージェントループ」です。モデルが次のアクション（ファイル読み込み、コード編集、コマンド実行など）を決定し、実行結果を確認し、次のアクションを決定する、というサイクルを繰り返します。テストが失敗した場合は原因を分析して修正を試み、成功するまでイテレーションを続けます。タスクの複雑さに応じて、完了までに1分〜30分程度かかります。

🌿 Worktree による並列安全性（Codex App）

2026年2月にリリースされたCodex Appでは、Gitのworktree機能を内蔵しています。複数のエージェントが同じリポジトリ上で作業する場合、それぞれが独立したコードのコピー上で動作するため、互いの変更が競合しません。異なるアプローチを並行して試すことも安全に行えます。

📋 AGENTS.md によるカスタマイズ

リポジトリに AGENTS.md というファイルを配置することで、Codexにプロジェクト固有の指示を与えられます。テストの実行コマンド、コーディング規約、プロジェクト構造の説明、避けるべきパターンなどを記述します。これはREADME.mdのような自然言語ファイルで、Codexが作業前に自動的に読み取ります。

Section 04 — How to Use

04使い方

Codex Cloudを使う方法は複数ありますが、最も基本的なのはChatGPTのWebインターフェースからの操作です。2026年3月現在、Codex App（デスクトップアプリ）、IDE拡張、GitHub上からの直接操作も可能になっています。

ChatGPTからの起動方法

Codexにアクセス

chatgpt.com/codex にアクセスするか、ChatGPTのサイドバーから「Codex」を選択します。初回はGitHubアカウントの連携が求められます。

リポジトリを選択

連携したGitHubアカウントからリポジトリを選択します。Codexはそのリポジトリのコード全体をサンドボックスにロードします。

タスクを入力

プロンプト欄にタスク内容を自然言語で記述します。コーディングタスクの場合は「Code」ボタン、コードベースに関する質問は「Ask」ボタンを押します。

進捗をリアルタイム監視

Codexの作業状況はリアルタイムで表示されます。ターミナルログ、編集中のファイル、テスト結果などを確認できます。GPT-5.3-Codexでは作業中にリアルタイムで会話・フィードバックも可能です。

結果をレビュー

タスク完了後、変更されたファイルのdiff、テスト結果、ターミナルログを確認します。問題がなければ「Open Pull Request」でGitHubにPRを作成するか、ローカル環境に変更を取り込みます。

GitHub連携

Codexは GitHub と深く統合されています。PR上で @codex をメンションすることで、Codexにコードレビューやタスク実行を依頼できます。IssueからCodexにタスクを割り当てることも可能です。リポジトリの自動レビュー設定を有効にすると、新しいPRに対して自動的にCodexレビューが実行されます。

効果的なタスク指示のコツ

✅ 良い指示の例

「src/api/users.ts に GET /users/:id エンドポイントを追加して。Prisma ORMでユーザーを取得し、存在しない場合は404を返す。ユニットテストも作成して。」

具体的スコープ明確テスト指定

❌ 避けるべき指示の例

「アプリをもっと良くして。」

曖昧スコープ不明成功基準なし

💡
プロのコツ：複数の小さなタスクに分割して並列実行すると、1つの大きなタスクを投げるよりも成功率が高くなります。「リファクタリング」「テスト追加」「ドキュメント更新」を別々のタスクとして同時に走らせましょう。

Section 07 — Security & Privacy

07セキュリティとプライバシー

企業でAIコーディングツールを採用する際、最大の懸念事項はセキュリティとプライバシーです。Codexはこれらの懸念に対して多層的なアプローチを取っています。

🔒 サンドボックス隔離

Codex Cloudの各タスクは、OpenAIが管理する隔離されたコンテナ内で実行されます。ホストシステムや無関係なデータへのアクセスは完全にブロックされます。2段階のランタイムモデルにより、セットアップ時に設定されたシークレット（APIキーなど）はエージェントフェーズ開始前に削除されます。

🌐 ネットワーク隔離

エージェントフェーズではデフォルトでインターネットアクセスが無効です。エージェントは明示的に提供されたGitHubリポジトリのコードと、セットアップスクリプトで事前インストールされた依存関係のみを利用できます。外部Webサイト、API、その他のサービスへのアクセスはできません。ユーザーが環境設定でインターネットアクセスを有効化することも可能ですが、プロンプトインジェクションのリスクが増加します。

🛡️ コード学習ポリシー

OpenAIのビジネスデータに関するポリシーとして、Business/Enterprise プランではデフォルトでユーザーのビジネスデータをモデルのトレーニングに使用しません。個人プラン（Free/Plus/Pro）では、設定によりオプトアウトが可能です。Enterprise向けにはデータ保持期間のカスタマイズやゼロデータ保持ポリシーも選択できます。

✅ 検証可能性（Verifiability）

Codexは実行したすべてのアクションの証跡を提供します。ターミナルログ、テスト出力、ファイル変更のdiffなど、各ステップを人間が追跡・検証できます。不確実な場合やテスト失敗時は、Codexが明示的にその旨を報告し、ユーザーが判断できるようにします。

🚫 悪用防止

Codexはマルウェア開発などの悪意ある用途を識別して拒否するよう訓練されています。一方で、カーネルエンジニアリングなどの正当なセキュリティ関連タスクは区別して許可します。GPT-5.3-Codexはサイバーセキュリティ能力が「高」に分類されており、一部の高リスクリクエストは自動的に前世代モデルにルーティングされる安全機構が導入されています。

📊 監査とモニタリング（Enterprise）

Enterprise向けにはOpenTelemetry（OTel）ベースのモニタリング、Compliance API による監査ログ、RBAC によるアクセス制御が提供されます。テレメトリはデフォルトOFFで、明示的に有効化する必要があります。プロンプト内容はデフォルトで秘匿化されます。

Section 08 — Use Cases

08ユースケース集 10選

Codexは幅広い開発タスクに活用できます。以下は実際の企業やチームでの活用例を含む代表的なユースケースです。

① テストカバレッジの向上

既存のコードベースに対してユニットテストを一括生成します。Superhuman社では、テストカバレッジの改善と統合テストの失敗修正にCodexを活用しています。「src/services/ 配下の全ファイルに対して、エッジケースを含むユニットテストを作成して」といった指示で、複数ファイルのテストを並列生成できます。

効果: テスト作成工数 80〜90% 削減

② 大規模リファクタリング

Temporal社では、大規模コードベースのリファクタリングにCodexを活用しています。例えば「deprecated APIの呼び出しをすべて新しいAPIに置き換える」「コールバックベースのコードをasync/awaitに変換する」といったリポジトリ全体にわたる変更を、複数エージェントで並列実行できます。

効果: リファクタリング工期 60〜80% 短縮

③ 新機能の実装スキャフォールド

新しい機能の骨格（ルーティング、コントローラー、モデル、テスト、型定義など）をCodexに生成させ、細部は人間が仕上げるワークフロー。Codexが「80%の基盤」を作り、開発者が「残り20%の品質」を担保します。

効果: 実装開始までの時間を 70% 短縮

④ オンコール問題のトリアージ

OpenAI社内では、エンジニアが朝一番にCodexにオンコール問題のトリアージを依頼するワークフローが定着しています。エラーログの分析、影響範囲の特定、修正パッチの提案まで、コンテキストスイッチを最小化しながらバグを処理できます。
