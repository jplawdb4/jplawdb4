1. 税理士業界のAI活用の現状（2025-2026年）

1.1 業界全体の動向

事実 インボイス制度（2023年10月～）と電子帳簿保存法の完全義務化（2024年1月～）により、確認すべき証憑の種類・ルールは複雑化し、実務の作業工数は増加した。一方で業界全体の人材不足は深刻化しており、「人を増やして対応する」従来手法だけでの解決は困難になっている。

事実 2025-2026年時点で、AIは「未来の技術」ではなく「日々の業務を支える実用的なツール」として定着しつつある。記帳代行は50～80%削減、申告書チェックは30～50%削減、レポート作成はほぼ自動化可能との報告がある。

1.3 freee自動仕訳の精度と限界

事実 freeeのAIデータ化サービスは、AI-OCRの活用とAIエージェントによる複数チェックにより、月間120仕訳の処理で記帳業務を4分の1（75%削減）に削減した実績がある。

推測 ただし「自動仕訳」は全件が100%正確というわけではなく、特に以下のケースでは人間の確認が必要と考えられる：

  • 初めての取引先・勘定科目の判定
  • 税区分の判定（軽減税率、非課税、不課税等の区別）
  • 複合仕訳（按分が必要なケース）
  • 期末の決算整理仕訳（減価償却、引当金等）

3.2「10社に1人のスタッフが必要」の妥当性

事実 監査担当者1人あたりの担当件数目安はおよそ20～30件。従業員1名あたりの平均売上は936万円。

推測 「10社に1人」は、フルサービス（記帳代行・月次監査・決算・申告）を提供する場合の保守的な目安としては妥当。ただし、これは「スタッフ（無資格の補助者）」の場合であり、税理士本人が担当する場合はより多くの件数をこなせる。クラウド会計の普及以前（紙伝票・手入力時代）の感覚に基づいている可能性が高い。

3.3 1人事務所で60社は通常あり得るか？

推測 以下の条件が揃えば、1人60社は達成可能と考えられる：

  • 顧客属性：IT/スタートアップ中心で取引パターンが単純（畠山氏の場合はこれに該当）
  • サービス範囲：記帳代行の大部分をAI化し、税理士は確認・修正に専念
  • ツール：freee等のクラウド会計 + AI仕訳の高精度活用
  • 顧問料設定：フルサービスではなく、効率化を前提とした価格体系

注意 ただし、60社全てが決算期を迎える繁忙期（特に3月決算→5月申告）の負荷、税務調査対応、臨時の相談対応などが集中した場合、1人での対応は極めて困難になるリスクがある。

「人件費3,000万円」の妥当性

推測 60社をスタッフで運営する場合の計算：

  • 60社 ÷ 10社/人 = 6人のスタッフが必要（「10社に1人」の基準）
  • スタッフ1人あたり年間人件費（給与＋社保＋福利厚生）= 約400万～550万円
  • 6人 × 500万円 = 3,000万円

事実 税理士事務所の人件費率は売上の約70%が「業界一給料の高い事務所」の水準。一般的には50-60%程度。

推測 スタッフ1人あたり平均年間人件費500万円×6人＝3,000万円という計算は、東京圏の税理士事務所スタッフの相場としてほぼ妥当。ただし「10社に1人」基準自体がやや保守的であり、20-30社/人基準なら2-3人（1,000-1,500万円）で足りる計算になる。

4. freee APIの活用事例

4.1 freee MCP（Model Context Protocol）

事実 freeeは2026年3月に「freee-mcp」をOSSとして公式公開。約270のAPIをMCPツール化し、Claude Desktop / Claude Code / Claude Cowork / Cursor等のAIエージェントから直接操作可能。

事実 対応業務：会計、人事労務、請求書発行、プロジェクト管理、販売管理の5領域。「請求書を作成して」とチャットで指示するだけで、取引先登録から請求書発行まで自動完了。

事実 税理士の米世毅事務所がnoteで「freee MCP × Claude Code：ゼロからはじめるセットアップガイド」等の実践記事を公開。

4.2 Claude Code × freee の具体的事例

事実 以下の自動化フローが実装・報告されている：

  1. Claude Codeで未処理明細を全件取得
  2. AIが勘定科目・取引先・品目・税区分を判定
  3. 自動登録ルールのCSVを生成
  4. freeeにCSVインポート
  5. 「自動で経理」画面で一括登録
  6. 消込も同時完了

事実 これにより「毎年1ヶ月ぐらいだらだらかかっていた経費整理が大幅に短縮できた」との報告あり。

4.3 ChatGPT向け「freee確定申告」アプリ

事実 freeeは2026年2月にChatGPT向けアプリ「freee確定申告」の提供を開始。税理士など専門家が回答した1万件以上の相談事例データベースから最適な情報を抽出して提示。AIの推測回答ではなく、実際の専門家回答に基づく。

4.4 その他のfreee API活用

  • 事実 「確定申告を自動化する Agent Skill "shinkoku"」がOSSとして公開（Zenn記事）
  • 事実 Claude Codeで「経理担当」を作った事例がnoteで公開
  • 事実 Claude × freee MCPで領収書をfreeeに自動登録した事例がnoteで報告

推測 freeeは日本の会計ソフト市場において、AIエージェント連携のエコシステム構築で他社をリードしている。MCP公開により、2026年以降はfreee + Claude/Cursorの組み合わせが税理士事務所のデファクトスタンダードになる可能性がある。

5. 税務業務のAI自動化の法的・倫理的論点

5.1 税理士法上の独占業務とAI

事実 税理士の独占業務（税理士法第52条）：

  1. 税務代理（申告、申請、不服申立て等の代理）
  2. 税務書類の作成（申告書、申請書等の作成）
  3. 税務相談（税務に関する相談への応答）

これらは無償であっても税理士以外の者が行うことはできない。

AIとの関係における論点整理

論点 現状の解釈 区分
AIが仕訳を自動生成 記帳代行は税理士の独占業務ではない。AIが仕訳を提案し、税理士が確認・承認するフローは合法 事実
AIが申告書ドラフトを作成 ドラフト作成はツールの利用であり、最終的な完成・提出の責任は税理士本人にある 事実
AIチャットボットが税務相談に回答 一般的な税務情報の提供は税務相談に該当しないが、個別具体的な税務判断の提示は税務相談に該当する可能性 推測
AIサービスが無人で税務処理 税理士が監督・最終確認する限り、ツールとしてのAI利用は現行法の枠内 推測

5.2 自動仕訳の責任所在

事実 AIがドラフトを作成できたとしても、最終的に完成・提出するのは税理士本人の責任。

推測 自動仕訳の誤りに起因する過少申告等が発生した場合、税理士の善管注意義務が問われる可能性がある。「AIが判断した」は免責事由にならない。

5.3 電子帳簿保存法との関係

事実 2024年1月から電子取引データの電子保存が完全義務化。保存要件：タイムスタンプ付与（最長2ヶ月+7営業日以内）、検索要件（取引年月日・金額・取引先の3項目）。

事実 AI-OCRを活用したサービス（OPTiM等）では、電帳法・インボイス制度の検索要件に必要な項目をAIが自動入力し、インボイス登録番号の有効性も自動判定する機能が実装されている。

推測 AI自動仕訳と電帳法は相性が良い。証憑のデジタル保存とAI読取を組み合わせることで、法令遵守と効率化を同時に達成できる。

5.4 守秘義務（税理士法38条）とクラウドAI

事実 税理士法第38条：「税理士は、正当な理由がなくて、税理士業務に関して知り得た秘密を他に洩らし、又は窃用してはならない。税理士でなくなった後においてもまた同様とする。」

クラウドAI利用時のリスクと対策

リスク 対策
入力データがAI学習に利用される可能性 事実 多くのサービスで「AI学習へのデータ提供拒否」オプトアウトが用意されている。利用規約の事前確認が必須
外部サーバーへの情報送信 事実 委託として正しく管理し、契約・保存場所・アクセス権限を整えれば、守秘義務違反にはならないとされる
データ漏洩リスク 推測 エンタープライズ版（Claude Team/Business, ChatGPT Enterprise等）の利用が望ましい
