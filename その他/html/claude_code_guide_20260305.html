Part 1 — Foundations / 基礎
Chapter 01 — What is Claude Code

Claude Codeとは

概要：ターミナルに住むAIペアプログラマー

Claude Code は、Anthropic社が開発・提供するエージェント型AIコーディングアシスタントです。ターミナル（コマンドライン）上で動作し、プロジェクトのコードベース全体を理解したうえで、ファイルの読み書き、シェルコマンドの実行、Git操作、テスト実行、PRの作成といった一連の開発タスクを自律的にこなします。単なるコード補完ツールとは異なり、Claude Codeは「エージェント」として振る舞います。つまり、開発者が高レベルの指示を与えれば、Claude Code自身が探索・計画・実装・検証のサイクルを回しながら、複数ファイルにまたがる複雑なタスクを遂行するのです。

Claude Codeは2025年2月にリサーチプレビューとして公開され、その後急速に進化を遂げました。ターミナルCLIとしてスタートしましたが、現在ではVS Code拡張、JetBrainsプラグイン、スタンドアロンのデスクトップアプリ、さらにはWebブラウザ上でのクラウド実行にも対応しており、あらゆる環境から同一のClaude Codeエンジンにアクセスできます。CLAUDE.mdファイル、settings.json、MCPサーバーの設定はすべてのサーフェスで共有されるため、環境間で一貫した体験が得られます。

Anthropic社の位置づけ

Anthropic社は、AI安全性研究を使命とするサンフランシスコ拠点の企業で、Claudeモデルファミリーの開発元です。Claude Codeは同社の「AIを安全かつ有用に」という理念を開発ツール分野に拡張したプロダクトであり、開発者が日常的に使用するCLI環境にClaude Opus 4.6やClaude Sonnet 4.6といった最先端モデルのパワーを直接持ち込みます。Claude 4モデルファミリーには現在、Claude Opus 4.6（最高性能）、Claude Opus 4.5、Claude Opus 4.1、Claude Opus 4、Claude Sonnet 4があり、Claude Codeはこれらのモデルをバックエンドとして利用します。

なぜCLI（ターミナル）なのか

多くのAIコーディングツールがIDE統合を前提として設計されている中、Claude Codeがターミナルを第一級のインターフェースとする理由は明確です。ターミナルはあらゆる開発環境のベースレイヤーであり、SSH経由のリモートサーバー、CI/CDパイプライン、コンテナ内部など、GUIが存在しない環境でも一貫して動作します。また、パイプやリダイレクトによるUnixツールとの組み合わせが自然に行え、自動化スクリプトへの組み込みも容易です。Claude Codeはこの設計哲学に基づき、「エディタ非依存でどこでも使えるAIコーディングパートナー」を実現しています。

Claude Codeの主要な特徴

コードベース全体の理解 — Claude Codeはプロジェクトのファイル構造、依存関係、コーディングパターンを自律的に読み取り、コンテキストを構築します。開発者が手動でファイルを選択・貼り付けする必要はありません。

エージェント型自律実行 — 指示を与えると、Claude Codeは自ら必要なファイルを読み、コマンドを実行し、コードを編集し、テストを走らせ、結果を検証します。この探索→計画→実装→検証のループを自動的に繰り返します。

権限ベースのセキュリティ — ファイル書き込みやシェルコマンド実行など、潜在的に危険な操作にはユーザーの承認が必要です。サンドボックス機能によるOS レベルの隔離も利用可能です。

マルチサーフェス対応 — ターミナルCLI、VS Code、JetBrains、デスクトップアプリ、Webブラウザのいずれからでも同じエンジンにアクセスでき、設定やコンテキストが共有されます。

拡張性 — MCP（Model Context Protocol）によるツール統合、カスタムスキル、フック、サブエージェント、プラグインにより、ほぼ無限に機能を拡張できます。

Chapter 02 — Installation & Setup

インストールと初期セットアップ

前提条件

Claude Codeを使用するには、Claude有料サブスクリプション（Pro、Max）またはAnthropic APIアカウントが必要です。対応プラットフォームはmacOS 13.0以降、Linux（glibc / musl）、Windows（Git for Windows必須）、およびWSL2です。メモリは最低4GB推奨で、低メモリ環境ではスワップ領域の追加が必要になる場合があります。

インストール方法

Anthropicはネイティブインストーラーを推奨しています。従来のnpmグローバルインストールも引き続き動作しますが、ネイティブインストーラーの方がバイナリサイズの最適化やOSレベルの統合が優れています。

macOS / Linux / WSL（推奨：ネイティブインストーラー）

curl -fsSL https://claude.ai/install.sh | bash

Windows PowerShell

irm https://claude.ai/install.ps1 | iex

Windows CMD

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

Homebrew（macOS / Linux）

brew install --cask claude-code

WinGet（Windows）

winget install Anthropic.ClaudeCode

初回起動と認証

インストール完了後、プロジェクトディレクトリに移動して claude と入力するだけでインタラクティブセッションが開始されます。初回起動時にはブラウザが開き、Claude.ai または Anthropic Console アカウントでのOAuth認証を求められます。認証が成功すると、ターミナルにプロンプトが表示され、即座に対話を開始できます。

cd your-project
claude

Tip：認証時にブラウザが自動で開かない場合は、ターミナルに表示されるURLをコピーして手動でブラウザに貼り付けてください。SSHリモート環境ではローカルマシンのブラウザでURLを開く必要があります。

認証コマンド

コマンド説明
claude auth loginログイン（--emailでメール指定、--ssoでSSO強制）
claude auth logoutログアウト
claude auth status認証状態をJSON出力（--textで可読形式）

アップデート

claude update

安定版チャネルを利用する場合は、settings.jsonで "autoUpdatesChannel": "stable" を設定します。stable版は通常最新リリースの約1週間後に配信され、重大なリグレッションのあるバージョンをスキップします。

Chapter 03 — Basic Usage

基本的な使い方

3つの基本モード

Claude Codeには、インタラクティブモード、ワンショットモード（print mode）、パイプ入力の3つの主要な利用パターンがあります。

インタラクティブモード

最も一般的な使い方です。claude と入力するだけで対話セッションが開始され、自然言語でプロンプトを入力しながらClaude Codeと会話できます。セッション中はファイル編集、テスト実行、Git操作などがリアルタイムに実行され、各操作に対する権限承認を求められます。

# 対話セッションを開始
claude

# 初回プロンプト付きで開始
claude "このプロジェクトの構造を説明して"

ワンショットモード（Print Mode / -p フラグ）

-p（--print）フラグを使うと、Claude Codeは質問に回答した後すぐに終了します。スクリプトや自動化パイプラインに組み込む場合に最適です。出力フォーマットとして text、json、stream-json を選択でき、後続のプログラムで構造化されたデータとして処理できます。

# テキスト出力
claude -p "この関数を説明して"

# JSON出力（スクリプト向け）
claude -p "全APIエンドポイントをリスト" --output-format json

# ストリーミングJSON
claude -p "このログファイルを分析" --output-format stream-json

パイプ入力

Unix的なパイプでデータをClaude Codeに渡すことができます。ビルドエラーの解析、ログファイルの要約、コードレビューの自動化などに活用できます。

# ビルドエラーの解析
cat build-error.txt | claude -p "このビルドエラーの根本原因を簡潔に説明して"

# コードレビュー
git diff main | claude -p "このdiffをレビューして問題点を指摘して"

# 出力をファイルに保存
cat logs.txt | claude -p "エラーパターンを分析" > analysis.txt

セッション管理

Claude Codeのセッションはローカルに保存されるため、前回の会話を再開できます。これはマルチタスクで複数の機能を並行開発する際に非常に便利です。

コマンド説明
claude -c現在のディレクトリで最新の会話を続行
claude -rセッションピッカーを表示 / セッション名で再開
claude --from-pr 123特定のPRに紐づくセッションを再開
/resumeセッション内でセッションピッカーを表示
/rename auth-migrationセッションに名前を付ける
/clearコンテキストをリセット（タスク切り替え時に推奨）
/compact会話履歴を要約してコンテキスト消費を削減

インタラクティブモードのキーボードショートカット

ショートカット操作
EscClaudeの実行を中断（コンテキストは保持）
Esc + Esc巻き戻しメニューを開く（会話・コード状態の復元）
Shift+Tab権限モードを切り替え（Normal → Auto-Accept → Plan）
Ctrl+Overbose モード切り替え（思考プロセス表示）
Option+T / Alt+T拡張思考のオン/オフ
Ctrl+B実行中のタスクをバックグラウンドに移動
@ファイル/ディレクトリをメンション

Chapter 04 — File Operations

ファイル操作 — 専用ツールの設計思想

Claude Codeは、ファイル操作に専用のツール群を備えています。単にBashの cat や sed を呼ぶのではなく、AIエージェントが確実かつ安全にファイルを扱えるよう設計された内部ツールが使われます。これにより、権限管理、変更追跡、エラーハンドリングが統一的に行われます。

Read — ファイル読み込み

指定パスのファイル内容を読み取ります。offset（開始行）と limit（読み取り行数）パラメータにより、巨大ファイルの一部のみを効率的に読み取ることもできます。Readは読み取り専用ツールであり、権限承認なしで実行されます。

# tool_input 例
{
  "file_path": "/path/to/file.txt",
  "offset": 10,
  "limit": 50
}

Edit — ファイル編集（文字列置換）

既存ファイル内の文字列を検索し、置換します。old_stringで検索対象、new_stringで置換後の内容を指定します。replace_all: true で全出現箇所を一括置換できます。Editは差分ベースで動作するため、ファイル全体を書き直すWriteよりも意図が明確で、レビューしやすい変更を生成します。

# tool_input 例
{
  "file_path": "/path/to/file.txt",
  "old_string": "function oldName()",
  "new_string": "function newName()",
  "replace_all": false
}

Write — ファイル書き込み（作成/上書き）

新規ファイルの作成、または既存ファイルの完全な上書きに使用されます。新しいファイルを生成する場合（テストファイルの追加など）や、ファイルの大部分を書き換える場合に使用されます。

Glob — ファイル検索（パターンマッチ）

globパターンでファイルを検索します。プロジェクト構造の探索や、特定の拡張子を持つファイルの一覧取得に使われます。path パラメータで検索開始ディレクトリを指定できます。

# TypeScriptファイルを検索
{ "pattern": "**/*.ts", "path": "/path/to/project/src" }

Grep — 内容検索（正規表現）

ファイル内容を正規表現で検索します。output_mode で出力形式を制御でき、files_with_matches（マッチしたファイル名のみ）、content（マッチ行を含む内容）、count（マッチ数）から選べます。大量ファイルの中から特定のAPI呼び出しやパターンを素早く見つけるのに最適です。

# TODO コメントの検索
{
  "pattern": "TODO.*fix",
  "path": "/path/to/project",
  "glob": "*.ts",
  "output_mode": "content"
}

設計思想：これらの専用ツールが存在する理由は、Bashコマンドに依存すると権限制御が困難になり、また出力のパースもエラーが起きやすくなるためです。専用ツールにより、Claudeは確定的な構造化結果を受け取り、より信頼性の高い操作が可能になります。

Chapter 05 — Bash Execution & Security Model

Bash実行とセキュリティモデル

Bashツール

Claude CodeのBashツールは、シェルコマンドを実行するための汎用ツールです。テストの実行、パッケージのインストール、Gitコマンド、ビルドスクリプトの実行など、ターミナルで行えるほぼすべての操作を委任できます。ただし、Bashコマンドはシステムに対して副作用を持つため、Claude Codeの中で最も厳格な権限管理の対象になっています。

権限モデル — 3層のアプローチ

Claude Codeは3種類のツールに対して異なる権限レベルを適用します。

ツール種別例承認「今後聞かない」の永続性
読み取り専用Read, Grep, Glob不要—
Bashコマンドシェル実行必要プロジェクト+コマンド単位で永続
ファイル変更Edit, Write必要セッション終了まで

権限モード

Claude Codeには5つの権限モードがあり、状況に応じた安全性と利便性のバランスを提供します。

モード説明主な用途
default各ツールの初回使用時に権限確認通常の開発
acceptEditsファイル編集を自動承認信頼できるコード変更作業
plan読み取り専用（コード変更・コマンド実行不可）コードベース探索、計画策定
dontAsk明示的に許可されたツール以外を自動拒否制限付きの自動化
bypassPermissions全権限チェックをスキップ隔離環境での完全自動化

サンドボックス

サンドボックスはOS レベルのファイルシステム・ネットワーク隔離を提供する機能です。有効化すると、Bashコマンドとその子プロセスがアクセスできるファイルパスとネットワークドメインが厳密に制限されます。これにより、プロンプトインジェクション等によってClaudeの意思決定がバイパスされた場合でも、重要なシステムリソースへのアクセスが物理的にブロックされます。

// .claude/settings.json
{
  "sandbox": {
    "enabled": true,
    "autoAllowBashIfSandboxed": true,
    "excludedCommands": ["git", "docker"],
    "filesystem": {
      "allowWrite": ["//tmp/build", "~/.kube"],
      "denyRead": ["~/.aws/credentials"]
    },
    "network": {
      "allowedDomains": ["github.com", "*.npmjs.org"],
      "allowLocalBinding": true
    }
  }
}

注意：--dangerously-skip-permissions フラグは全権限チェックをスキップします。CI/CDパイプラインなど完全に隔離された環境でのみ使用してください。管理者は disableBypassPermissionsMode: "disable" で組織全体でこのフラグを無効化できます。

権限ルール構文

settings.jsonの permissions セクションで、allow、ask、deny のルールを定義できます。ルールは deny → ask → allow の順に評価され、最初にマッチしたルールが適用されます。

{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git commit *)",
      "Read(~/.zshrc)"
    ],
    "deny": [
      "Bash(curl *)",
      "Read(./.env)",
      "Read(./secrets/**)"
    ]
  }
}
Chapter 06 — Git Integration

Git統合

Claude CodeはGitネイティブ

Claude Codeは設計レベルでGitと深く統合されています。Gitリポジトリ内で起動すると、ブランチ情報、コミット履歴、差分を自動的に活用し、コンテキストに応じた操作を行います。gh（GitHub CLI）がインストールされている場合、PRの作成、イシューの参照、コメントの読み取りなどの操作もシームレスに行えます。

基本的なGitワークフロー

Claude Codeに自然言語で指示するだけで、Gitの操作を一連のフローとして実行させることができます。

コミット

# 変更内容を確認してコミット
> 変更内容を確認して、意味のあるコミットメッセージでコミットして

# Conventional Commitsスタイル
> Conventional Commits形式でコミットメッセージを作って

Claude Codeはデフォルトで Co-Authored-By: Claude トレーラーをコミットメッセージに付与します。この挙動は attribution 設定でカスタマイズまたは無効化できます。

PR作成

> 変更内容のPRを作成して。説明も書いて

# Claude Code が以下を自動実行:
# 1. git diff で変更を確認
# 2. 意味のあるブランチ名を生成
# 3. コミットメッセージを作成
# 4. gh pr create でPRを作成

gh pr create で作成されたPRは自動的にセッションと紐付けられ、後から claude --from-pr 123 で再開できます。

Git Worktree による並列作業

--worktree（-w）フラグを使うと、独立したGit worktreeにClaude Codeを起動できます。これにより、メインの作業ディレクトリに影響を与えることなく、複数のClaude Codeセッションを並列に実行できます。

# feature-auth というworktreeで起動
claude --worktree feature-auth

# 自動生成された名前でworktreeを作成
claude --worktree

# 別のセッション
claude --worktree bugfix-123

worktreeは <repo>/.claude/worktrees/<name> に作成されます。セッション終了時に変更がなければ自動的にクリーンアップされ、変更がある場合はClaude Codeが保持するか削除するか確認します。

GitHub Actions統合

Claude Code GitHub Actionsを使えば、PRやイシューで @claude とメンションするだけでClaude Codeによる自動分析・コードレビュー・修正提案を受けられます。GitLab CI/CDにも対応しており、同様のワークフローを構築できます。

ベストプラクティス：Claude Codeでのコミットは小さく、焦点を絞ったものにしましょう。大きなコミットよりも、アトミックな変更の方がClaudeがより正確なコミットメッセージを生成でき、後から /rewind で巻き戻しやすくなります。

Chapter 07 — CLAUDE.md

CLAUDE.md — プロジェクト設定・メモリ・カスタム指示

CLAUDE.mdとは

CLAUDE.mdは、Claude Codeがセッション開始時に自動的に読み込む特別なマークダウンファイルです。プロジェクト固有のルール、コーディング規約、ワークフロー、環境変数の説明などを記載しておくことで、Claudeが毎回のセッションでこれらのコンテキストを持った状態で作業を開始します。「AIエンジニアのための .editorconfig + README」とも言える存在です。

配置場所と優先順位

場所スコープ共有
~/.claude/CLAUDE.md全プロジェクト共通（個人）いいえ
./CLAUDE.md or .claude/CLAUDE.mdプロジェクトルートはい（Git管理推奨）
CLAUDE.local.mdプロジェクトローカル（個人）いいえ（.gitignore）
親ディレクトリの CLAUDE.mdモノレポ構造で自動継承はい
子ディレクトリの CLAUDE.md作業時にオンデマンド読み込みはい

効果的なCLAUDE.mdの書き方

CLAUDE.mdは毎セッション読み込まれるため、簡潔かつ実用的な内容にすることが重要です。/init コマンドでプロジェクトを分析し、ビルドシステム・テストフレームワーク・コードパターンを自動検出した初期ファイルを生成できます。

# コードスタイル
- ES modules (import/export) を使用、CommonJS (require) は使わない
- インポートは可能な限りデストラクチャリング

# ワークフロー
- コード変更後は必ず型チェックを実行
- テストは全体ではなく個別のテストファイルを実行（パフォーマンス重視）

# プロジェクト固有
- APIエンドポイントは /api/v2/ プレフィックスを使用
- エラーレスポンスは { error: { code, message } } 形式

含めるべき情報 / 含めるべきでない情報

含めるべき含めるべきでない
Claudeが推測できないBashコマンドコードを読めば分かること

Chapter 08 — MCP Integration

MCP（Model Context Protocol）サーバー統合

MCPとは

MCP（Model Context Protocol）は、AIツール統合のためのオープンソース標準プロトコルです。Anthropicが提唱し、現在は幅広いエコシステムで採用されています。MCPサーバーをClaude Codeに接続することで、データベースへのクエリ、イシュートラッカーとの連携、モニタリングデータの分析、Figmaデザインの統合、SaaSツールとの自動化など、Claude Codeの能力を大幅に拡張できます。

MCPで実現できること

イシュートラッカー連携

「JIRA の ENG-4521 に記載された機能を実装して PRを作って」

データベースクエリ

「PostgreSQL で過去90日間購入のないユーザーを検索して」

モニタリング分析

「Sentry で過去24時間の頻出エラーを確認して」

デザイン統合

「Figma の新デザインに基づいてメールテンプレートを更新して」

MCPサーバーの追加方法

リモート HTTP サーバー（推奨）

# Notion 接続
claude mcp add --transport http notion https://mcp.notion.com/mcp

# 認証ヘッダー付き
claude mcp add --transport http secure-api https://api.example.com/mcp \
  --header "Authorization: Bearer your-token"

ローカル stdio サーバー

# Airtable サーバー
claude mcp add --transport stdio --env AIRTABLE_API_KEY=YOUR_KEY airtable \
  -- npx -y airtable-mcp-server

MCPサーバーのスコープ

スコープ保存先用途
Local（デフォルト）~/.claude.json（プロジェクト別）個人・実験的設定
Project.mcp.json（リポジトリルート）チーム共有（Git管理）
User~/.claude.json（グローバル）全プロジェクト共通の個人ツール

MCPサーバーの管理

# 一覧表示
claude mcp list

# 詳細確認
claude mcp get github

# 削除
claude mcp remove github

# セッション内でステータス確認
/mcp

MCP Tool Search

多数のMCPサーバーを接続すると、ツール定義がコンテキストウィンドウの大部分を消費する問題が生じます。MCP Tool Searchはこの問題を解決する機能で、ツール定義がコンテキストの10%を超えると自動的に有効化されます。有効化されると、MCPツールは事前読み込みされず、Claudeが必要に応じて検索ツールを使って動的に発見・ロードします。

Chapter 09 — Multi-Agent & Teams

マルチエージェント・チーム機能

サブエージェント（Subagents）

サブエージェントは、独自のコンテキストウィンドウ、システムプロンプト、ツールアクセス権を持つ特化型AIアシスタントです。メインの会話から特定のタスクを委任し、その結果だけを受け取ることで、メインコンテキストの消費を抑えつつ専門的な作業を行えます。

組み込みサブエージェント

名前モデルツール用途

カスタムサブエージェントの作成

Markdownファイル（YAMLフロントマター付き）としてサブエージェントを定義できます。

# .claude/agents/security-reviewer.md
---
name: security-reviewer
description: コードのセキュリティ脆弱性をレビュー
tools: Read, Grep, Glob, Bash
model: opus
---

あなたはシニアセキュリティエンジニアです。以下の観点でコードをレビューしてください：
- インジェクション脆弱性（SQL, XSS, コマンドインジェクション）
- 認証・認可の欠陥
- コード内のシークレットやクレデンシャル
- 安全でないデータハンドリング

具体的な行参照と修正提案を含めてください。

Agent Teams（エージェントチーム）

Agent Teamsは、複数のClaude Codeインスタンスが協調して作業する機能です。1つのセッションが「チームリード」として機能し、タスクの割り当て・進捗の統合を行います。チームメイトはそれぞれ独立したコンテキストウィンドウを持ち、ピアツーピアで直接通信できます。サブエージェントとの最大の違いは、チームメイト同士がメッセージを交換し、知見を共有し、互いの成果物を検証できる点です。

Agent Teamsの有効化

Chapter 10 — Custom Skills

カスタムスキル — 独自ワークフローの定義

スキルとは

スキルは、Claude Codeの知識と能力を拡張するための仕組みです。SKILL.mdファイルに指示を記述すると、Claudeがそれをツールキットに追加します。Claudeは関連するコンテキストでスキルを自動的に適用するか、/skill-nameで直接呼び出すことができます。スキルはAgent Skillsオープン標準に準拠しています。

スキルの種類

スキルファイルは大きく「リファレンスコンテンツ」と「タスクコンテンツ」の2種類に分けられます。リファレンスコンテンツは規約やパターンなどの知識を提供し、Claudeがインラインで参照します。タスクコンテンツはデプロイメントやコミットなどの具体的なアクション手順を提供し、通常 /skill-name で明示的に呼び出します。

バンドルスキル

スキル説明
/simplify最近変更したファイルを3つの観点（コード再利用、品質、効率）で並列レビューし修正
/batch <instruction>大規模な変更を5〜30の独立した作業単位に分解し、worktreeごとに並列実行
/debug [description]現在のセッションのデバッグログを読んでトラブルシュート

カスタムスキルの作成例

# .claude/skills/fix-issue/SKILL.md
---
name: fix-issue
description: GitHub issueを修正する
disable-model-invocation: true
---
GitHub issue $ARGUMENTS を分析して修正してください。

1. `gh issue view` でissueの詳細を取得
2. 問題を理解
3. コードベースで関連ファイルを検索
4. 修正を実装
5. テストを書いて実行
6. リンティングと型チェックを通過させる
7. 記述的なコミットメッセージを作成
8. プッシュしてPRを作成

実行: /fix-issue 1234

フロントマター設定

フィールド説明
nameスキルの識別名
descriptionClaudeがいつ使うか判断するための説明
disable-model-invocationtrueでClaude自動呼出しを禁止（手動 /nameのみ）
user-invocablefalseで/メニューから非表示（Claude専用）
allowed-toolsスキル実行中に使用可能なツールを制限
contextforkでサブエージェントコンテキストで実行
agentcontext: fork時に使用するサブエージェント種別
Chapter 11 — Hooks

フック — イベント駆動の自動化

フックとは

フックは、Claude Codeのライフサイクルの特定のポイントで自動的に実行されるシェルコマンド、HTTPエンドポイント、またはLLMプロンプトです。CLAUDE.mdの指示が「助言的」であるのに対し、フックは「決定的」であり、アクションの確実な実行を保証します。

フックイベント一覧

イベント発火タイミングブロック可
SessionStartセッション開始・再開時—
UserPromptSubmitプロンプト送信時（処理前）✓
PreToolUseツール実行前✓
PostToolUseツール成功後—
PostToolUseFailureツール失敗後—
PermissionRequest権限ダイアログ表示時✓
StopClaudeの応答完了時✓
Notification通知送信時—
SubagentStart / Stopサブエージェント開始/終了—/✓
TeammateIdleチームメイトがアイドル状態に✓
TaskCompletedタスク完了マーク時✓
ConfigChange設定ファイル変更時✓
PreCompactコンテキスト圧縮前—
SessionEndセッション終了時—

フック設定例：危険なコマンドのブロック

// .claude/settings.json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/block-rm.sh"
          }
        ]
      }
    ]
  }
}
#!/bin/bash
# .claude/hooks/block-rm.sh
COMMAND=$(jq -r '.tool_input.command')

if echo "$COMMAND" | grep -q 'rm -rf'; then
  jq -n '{
    hookSpecificOutput: {
      hookEventName: "PreToolUse",
      permissionDecision: "deny",
      permissionDecisionReason: "破壊的コマンドがフックによりブロックされました"
    }
  }'
else
  exit 0
fi

フックハンドラーの種類

種類説明
commandシェルコマンド実行（JSON入力をstdinで受信）
httpHTTPエンドポイントにPOST送信
promptLLMモデルに単発評価を依頼
agentツール使用可能なサブエージェントを起動して検証

終了コードの意味

コマンドフックの終了コードによって動作が決まります。exit 0 は成功（続行）、exit 2 はブロッキングエラー（ツール呼び出しの阻止やプロンプトの拒否）、その他の終了コードは非ブロッキングエラー（verboseモードでstderrを表示し、実行は続行）です。

Chapter 12 — IDE Integration & Headless Mode

IDE統合とヘッドレスモード

VS Code拡張

Claude Code VS Code拡張（anthropic.claude-code）は、インラインDiff、@メンション、プランレビュー、会話履歴をエディタ内に直接統合します。拡張機能ビュー（Cmd+Shift+X）で「Claude Code」を検索してインストールするか、Command Palette（Cmd+Shift+P）から「Claude Code: Open in New Tab」を選択して起動します。Cursor エディタにも同じ拡張がインストール可能です。

JetBrains プラグイン

IntelliJ IDEA、PyCharm、WebStormなどのJetBrains IDEに対応するプラグインが JetBrains Marketplace で提供されています。インタラクティブなDiffビューイングや選択コンテキストの共有が可能です。

デスクトップアプリ

スタンドアロンのデスクトップアプリケーションを使えば、IDEやターミナルの外でClaude Codeを実行できます。Diffの視覚的なレビュー、複数セッションの並列実行、クラウドセッションの起動が可能です。macOS（Intel/Apple Silicon）とWindows（x64/ARM64）に対応しています。

Web（claude.ai/code）

ブラウザ上でClaude Codeを実行できます。ローカルセットアップ不要で、長時間実行タスクを開始して後から確認したり、ローカルにないリポジトリで作業したり、複数タスクを並列に実行したりできます。Claude iOSアプリからも利用可能です。

ヘッドレスモード（非対話モード）

claude -p を使用するヘッドレスモードは、CI/CDパイプライン、プリコミットフック、自動化ワークフローへの組み込みに最適です。

# CI/CDパイプラインでの使用例
claude -p "lint結果を確認して問題を修正" --output-format json

# ストリーミング出力
claude -p "テストを実行して結果を報告" --output-format stream-json

# 予算制限付き
claude -p "コードレビューを実行" --max-budget-usd 5.00

# ターン数制限
claude -p "型エラーを修正" --max-turns 3

構造化出力（JSON Schema）

--json-schema フラグを使うと、Claude Codeのワークフロー完了後にJSON Schemaに準拠した構造化出力を取得できます。

claude -p "このファイルの関数一覧を返して" \
  --json-schema '{"type":"object","properties":{"functions":{"type":"array","items":{"type":"string"}}}}'

3つの利用方法

Claude Codeの料金体系は、サブスクリプションプラン（Claude Pro / Max）とAPI従量課金の2つに大別されます。

プラン月額概要

Claude Max 5x$100Pro の5倍の使用量。Opus アクセス、ピーク時優先
Claude Max 20x$200Pro の20倍の使用量。ヘビーユーザー向け
API（従量課金）使用量に応じた課金完全なコスト可視性。高コストになりやすい

APIトークン料金（主要モデル）

モデル入力 ($/1M tokens)出力 ($/1M tokens)キャッシュ読取 ($/1M)キャッシュ書込 ($/1M)

Claude Sonnet 4$3$15$0.30$3.75

コスト最適化のポイント

Claude Codeの使用トークンの90%以上はキャッシュ読み取りが占めます。これはClaude Codeが常にコードベースのコンテキストをキャッシュしているためです。実際のユーザー報告では、月間10億トークン以上を使用してAPI換算で$5,000以上に相当する使用量を、Max 5xプラン（$100/月）で賄えたケースがあります。日常的にClaude Codeを開発の中心ツールとして使う場合、Max プランは API と比較して大幅なコスト削減になります。

損益分岐点の目安：月間のAPI相当コストが$100を超える場合はMax 5x、$200を超える場合はMax 20xが経済的です。軽い使用（月50Mトークン未満）であればProまたはAPIが適しています。

コスト削減テクニック

/compact でコンテキストを定期的に圧縮する。無関係なタスク間で /clear を使う。Exploreサブエージェント（Haikuモデル）を活用してメインコンテキストの消費を抑える。サンドボックスを有効化して autoAllowBashIfSandboxed で権限プロンプトを削減する。ステータスラインでリアルタイムにコンテキスト使用率を監視する。これらの手法を組み合わせることで、同じ予算でより多くの作業を遂行できます。

Chapter 14 — Use Cases

ユースケース集（15選）

1. コードベースの理解

新しいプロジェクトに参加した際に「このプロジェクトの構造を説明して」と聞くだけで、アーキテクチャ、主要モジュール、技術スタックの概要を即座に把握できます。

レガシーコードのモダン化、パターンの統一、型安全性の向上などを、プロジェクト全体の一貫性を保ちながら実行します。

4. テスト作成

既存のテストパターンを分析し、エッジケースを含むテストを自動生成。テストランナーの実行まで一貫して行います。

5. PR作成とレビュー

変更内容の分析、ブランチ作成、コミット、PR説明の記述までを自動化。GitHub Actions連携で自動レビューも可能。

6. ドキュメント生成

コードからAPIドキュメント、READMEの更新、JSDocコメントの追加などを自動生成します。

7. CI/CDパイプライン統合

ヘッドレスモードでリンター、コードレビュー、テスト生成をCIに組み込み、プッシュごとに自動チェックを実行します。

8. データベースクエリ分析

MCPでPostgreSQLに接続し、自然言語で「今月の総売上は？」と聞くだけでSQLクエリを生成・実行します。

9. 大規模マイグレーション

/batchスキルで「全コンポーネントをReactからVueに移行」のような大規模変更を並列worktreeで実行します。

10. セキュリティレビュー

カスタムサブエージェントにセキュリティ専門の指示を与え、コード全体の脆弱性スキャンを実行します。

11. パフォーマンス最適化

プロファイリングデータを分析し、ボトルネックの特定と最適化提案を行います。

12. 環境構築の自動化

新しいチームメンバーのオンボーディング手順をスキルとして定義し、環境セットアップを自動化します。

13. Slackからバグ報告→PR

Slack統合でバグ報告を受け取り、自動的にイシュー作成、修正実装、PR作成まで行います。

14. Webアプリのデバッグ

Chrome拡張を使って実際のブラウザ上でUIの動作を確認し、問題を特定・修正します。

15. チームコードレビュー

Agent Teamsで3人のレビュアー（セキュリティ、パフォーマンス、テストカバレッジ）を並列起動し、多角的なレビューを実施します。

settings.json 完全リファレンス

設定ファイルの場所

スコープ場所共有
ユーザー~/.claude/settings.jsonいいえ
プロジェクト.claude/settings.jsonはい（Git管理）
ローカル.claude/settings.local.jsonいいえ（gitignore）
管理者OS別のシステムパスはい（IT管理）

主要設定キー

キー型説明
modelstringデフォルトモデル（例: "claude-sonnet-4-6"）
permissionsobjectallow, ask, deny ルール配列
permissions.defaultModestringデフォルト権限モード
sandboxobjectサンドボックス設定（enabled, ファイルシステム, ネットワーク）
hooksobjectライフサイクルフック定義
envobject環境変数の設定
attributionobjectGit コミット/PR の帰属表示カスタマイズ
languagestring応答言語（例: "japanese"）
outputStylestring出力スタイル調整
cleanupPeriodDaysnumber非アクティブセッション削除期間（デフォルト30日）
alwaysThinkingEnabledboolean拡張思考をデフォルト有効化
autoUpdatesChannelstring"stable" or "latest"
enabledPluginsobjectプラグイン有効化設定
teammateModestringAgent Teams表示モード（"auto", "in-process", "tmux"）
companyAnnouncementsarray起動時の組織アナウンスメント

{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "model": "claude-sonnet-4-6",
  "language": "japanese",
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test *)",

      "Read(./.env)",
      "Read(./secrets/**)"

  },

      }
    ]

Appendix C — Troubleshooting

よくあるトラブルと解決法

問題原因解決策
command not found: claudePATHにインストールディレクトリが含まれていない~/.local/bin をPATHに追加。echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
インストール時に Killedメモリ不足（OOM Killer）2GBのスワップを追加: sudo fallocate -l 2G /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile
TLS/SSL接続エラーCA証明書の問題 / 企業プロキシsudo apt-get install ca-certificates / NODE_EXTRA_CA_CERTS 設定
OAuth認証エラーコードの期限切れ / ブラウザ問題Enterで再試行 / cキーでURLをコピーして手動ブラウザ
403 Forbiddenサブスクリプション切れ / プロキシ干渉claude.ai/settings でプラン確認 / プロキシ設定の見直し
検索が動作しないripgrepがインストールされていないbrew install ripgrep (macOS) / apt install ripgrep (Ubuntu)
JetBrains IDEが検出されない（WSL2）WSL2ネットワーク/ファイアウォールWindows Firewallルール追加 or .wslconfig でmirrored networking
高CPU/メモリ使用巨大なコンテキスト/compact で圧縮、タスク間で /clear
権限プロンプトが頻繁許可ルール未設定/permissions で頻用コマンドを許可リストに追加
コンテキストが溢れる長時間セッションサブエージェントに調査を委任、/clear でリセット
