# tsutatsu — 通達DB quickstart

base = https://jplawdb4.github.io/jplawdb4/

## 直接アクセス（推奨・1回で完結）

URL: `{base}tsutatsu/text/{doc_code}/{item_id}.txt`

各通達は平均1-2KB（0.3-0.5Kトークン）。10通達読んでも5Kトークン以下。

例:
- 法基通9-4-1  → `tsutatsu/text/hojinzei_kihon_tsutatsu/9-4-1.txt`
- 所基通33-1   → `tsutatsu/text/shotokuzei_kihon_tsutatsu/33-1.txt`
- 消基通5-1-1  → `tsutatsu/text/shohizei_kihon_tsutatsu/5-1-1.txt`
- 相基通1-2-1  → `tsutatsu/text/sozokuzei_kihon_tsutatsu/1-2-1.txt`
- 評基通6      → `tsutatsu/text/hyoka_kihon_tsutatsu/6.txt`
- 措通66の6-1  → `tsutatsu/text/sozei_tokubetsu_tsutatsu_hojinzei/66-6-1.txt`
- 通法基通1-2  → `tsutatsu/text/kokuzei_tsusoku_kihon_tsutatsu/1-2.txt`

## 番号の変換ルール

item_idには2種類の区切り文字がある。意味が異なるので混同しないこと。

**`-`（ハイフン）= 「の」（階層）**
- 「の」→ `-`（例: 措通66の6-1 → `66-6-1`）
- 漢数字→算用数字（「第九章第四節」ではなく `9-4-1`）
- サブ通達: `1-3-2-1`（4階層まで）

**`_`（アンダースコア）= 「・」（並列・共通通達）**
- 複数条文にまたがる共通通達で使用（58件）
- `common` = 「共」

| 通達表記 | item_id | 説明 |
|---------|---------|------|
| 所基通9-4-1 | `9-4-1` | 通常（ハイフンのみ） |
| 措通66の6-1 | `66-6-1` | 「の」→ `-` |
| 所基通36・37共-19 | `36_37_common-19` | 「・」→ `_`、「共」→ `common` |
| 所基通194・195-1 | `194_195-1` | 「・」→ `_`（commonなし） |
| 相基通1の3・1の4共-1 | `1-3_1-4_common-1` | 「の」→ `-` と「・」→ `_` の混在 |
| 所基通37-10・37-11共-1 | `37-10_37-11_common-1` | 同上 |

**よくある間違い:**
- ❌ `36-37-common-19` → ⭕ `36_37_common-19`（「・」は `_` であって `-` ではない）
- ❌ `1-3-1-4-common-1` → ⭕ `1-3_1-4_common-1`（「1の3」と「1の4」の境界は `_`）

**措通(法)の特徴**: item_idが措置法の条文番号と直接対応
- 措通42-6-1 → 措置法42条の6に関する通達第1号
- 措通66-6-1 → 措置法66条の6に関する通達第1号

## 番号が不明な場合

`tsutatsu/data/resolve_lite/{doc_code}.json` で全項番+タイトル一覧を取得（1回）

### 使い方
1) 条文番号がある: SECTION_INDEX の refs 列を検索して、該当する section を得る
2) キーワードがある: SECTION_INDEX の label 列を検索して、該当する section を得る
3) range（例: 9-2-1..9-2-52）内の item_id をそのまま text に使う

略記: 法=法人税法 / 令=法人税法施行令 / 規=法人税法施行規則 / 措法=租税特別措置法 / 措令=措置法施行令
